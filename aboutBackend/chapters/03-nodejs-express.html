하 · 입문

한 줄로 이해하기

Node.js는 자바스크립트로 서버를 만들 수 있게 해주는 실행 환경이고, Express는 그 위에서 빠르게 웹 서버를 작성하도록 도와주는 프레임워크입니다.

왜 배워야 하나?

당근마켓, 토스, 넷플릭스 같은 서비스 일부가 Node.js로 운영됩니다. 자바스크립트 한 언어로 프론트엔드와 백엔드를 모두 다룰 수 있어 작은 팀에서도 빠르게 풀스택 서비스를 만들 수 있습니다. 챗봇 같은 실시간 양방향 통신 서비스나 IoT 기기 제어 서버처럼 동시 연결이 많은 시스템에서 특히 강합니다. Node.js와 Express를 익히면 npm 생태계의 막대한 라이브러리를 활용해 짧은 시간에 프로토타입을 완성할 수 있습니다.

🌱 핵심 용어

  • Node.js: V8 엔진 위에서 자바스크립트를 실행하는 런타임.
  • npm: Node.js용 패키지 매니저(라이브러리 설치 도구).
  • Express: 라우팅과 미들웨어를 단순화한 웹 프레임워크.
  • 모듈: 기능을 파일 단위로 나누어 require/import로 불러오는 단위.

개념 풀어쓰기

Node.js를 자동차에 비유한다면 V8 엔진이 자동차의 엔진, Node.js가 차체와 변속기, 그리고 Express는 운전을 쉽게 해주는 내비게이션 같은 보조 시스템입니다. 자바스크립트는 원래 브라우저에서 화면을 다루는 언어였지만, Node.js 덕분에 파일 시스템, 네트워크, 데이터베이스도 다룰 수 있게 되었습니다. Express는 "GET /users 요청이 들어오면 이 함수를 실행하라" 같은 규칙을 짧은 코드로 작성하게 해줍니다. 또한 Express의 미들웨어 개념은 요청과 응답 사이에 끼어들어 인증, 로깅, 데이터 변환을 처리하는 컨베이어 벨트 같은 구조로, 코드 재사용성을 크게 높여줍니다.

💡 학습 TIP: npm init -y로 프로젝트를 만들고, npm install express 한 번이면 시작할 수 있습니다. 5줄로 서버를 띄우는 것부터 해보세요.

요약

  • Node.js는 자바스크립트로 서버를 만드는 런타임이다.
  • Express는 라우팅과 미들웨어를 쉽게 다룬다.
  • npm 생태계로 다양한 라이브러리를 활용한다.
  • 비동기 I/O에 강해 동시 연결 처리에 유리하다.
  • 풀스택을 한 언어로 작성할 수 있다.
중 · 응용

이론 깊이

Node.js는 V8 엔진 위에 libuv를 두고 단일 스레드 이벤트 루프를 사용합니다. 이벤트 루프는 timers, pending callbacks, idle/prepare, poll, check, close 단계로 순환합니다. 비동기 I/O는 libuv의 스레드 풀(기본 4개)에서 처리되어 메인 스레드를 막지 않습니다. CommonJS(require)와 ES Modules(import) 두 모듈 시스템이 공존합니다. Express는 미들웨어 체인 모델을 따릅니다. app.use(fn)으로 등록된 함수들이 (req, res, next) 시그니처를 가지며 next()로 다음 미들웨어로 흐름을 넘깁니다. 라우터는 모듈 단위로 분리할 수 있어 큰 앱에서 유지보수가 쉽습니다. body-parser, cors, helmet 같은 표준 미들웨어로 보안과 파싱을 덧붙입니다. 에러는 (err, req, res, next) 4인자 미들웨어로 모아 처리합니다. 요즘은 Fastify, NestJS, Koa 같은 대안도 많이 쓰입니다. NestJS는 Angular 영감을 받은 모듈/DI 구조로 대규모 프로젝트에 적합합니다.

코드 패턴

// Express 기본 앱
const express = require('express');
const app = express();
app.use(express.json());
app.get('/health', (req, res) => res.json({ ok: true }));
app.use((err, req, res, next) => {
  console.error(err);
  res.status(500).json({ error: 'Internal Server Error' });
});
app.listen(3000);

💻 예제 1: 라우터 분리

const router = express.Router(); router.get('/', list); 으로 모듈화하고 app.use('/users', userRouter)로 마운트합니다.

💻 예제 2: async 핸들러

async (req, res) => {} 안에서 throw하면 4인자 에러 미들웨어로 자동 전달되도록 express-async-errors를 도입합니다.

실무 응용

실시간 채팅 서비스는 Node.js + Socket.IO로 양방향 통신을 구현합니다. 결제 시스템 같은 트랜잭션이 중요한 서비스는 NestJS의 트랜잭션 데코레이터로 명확히 경계를 정의합니다. 대용량 로그 수집기는 Fastify의 낮은 오버헤드를 활용해 초당 수만 요청을 처리합니다.

⚠️ 자주 하는 실수: 이벤트 루프 안에서 동기 암호화나 거대 JSON.parse를 호출해 전체 서버를 멈추게 만드는 경우가 흔합니다. CPU 작업은 worker_threads로 옮기세요.
상 · 심화

전문가 레벨

대규모 Node.js 시스템에서는 단일 프로세스의 한계를 cluster 모듈이나 PM2의 멀티 프로세스, 또는 Kubernetes의 수평 확장으로 극복합니다. V8 엔진은 hidden class와 인라인 캐시로 자바스크립트 객체 접근을 최적화하므로, 동일한 형태의 객체를 일관되게 사용해야 JIT 최적화 혜택을 받습니다. GC는 세대별 수집(young/old generation)이며 large object는 직접 old generation으로 가서 GC 압박을 늘리니 버퍼 재사용 풀이 도움 됩니다. async_hooks API로 비동기 컨텍스트를 추적해 분산 트레이싱 컨텍스트를 자연스럽게 전파할 수 있습니다. AsyncLocalStorage는 요청별 컨텍스트(예: 요청 ID, 유저 ID)를 깔끔하게 관리합니다. Stream API는 백프레셔를 통해 메모리 폭증 없이 대용량 데이터를 처리합니다. 네이티브 모듈은 N-API로 작성해 Node 버전 간 ABI 호환성을 보장합니다. 보안 측면에서는 prototype pollution, command injection, dependency confusion 공격을 정기적으로 점검하고 npm audit, snyk 같은 도구로 의존성을 감시합니다. 메모리 누수는 heap snapshot diff와 clinic.js로 진단하며, CPU 프로파일은 0x flame graph로 시각화합니다.

고급 패턴

// AsyncLocalStorage 요청 컨텍스트
const { AsyncLocalStorage } = require('async_hooks');
const als = new AsyncLocalStorage();
app.use((req, res, next) => {
  als.run({ requestId: req.headers['x-request-id'] }, next);
});
function logWithContext(msg) {
  const ctx = als.getStore();
  console.log(JSON.stringify({ msg, requestId: ctx?.requestId }));
}

최신 동향

Node.js 20+의 내장 테스트 러너와 watch 모드는 외부 도구 의존을 줄여줍니다. Bun, Deno 같은 대안 런타임은 더 빠른 시작과 내장 TypeScript를 제공합니다. NestJS는 마이크로서비스 트랜스포트(Kafka, NATS, Redis)를 추상화해 서비스 메시 환경에서 잘 어울립니다. 엣지 런타임(Cloudflare Workers, Vercel Edge)은 Node 호환 API 일부를 제공해 동일한 코드를 글로벌 분산 실행할 수 있게 합니다. Permission Model(--permission 플래그)은 Deno와 유사한 권한 기반 보안을 Node에 도입하고 있습니다.
