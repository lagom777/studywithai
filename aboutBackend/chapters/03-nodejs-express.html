Chapter 3 — Node.js Runtime & Express Framework

이벤트 루프 3D 시각화 포함

Node.js란?

Node.js는 Chrome V8 엔진 기반의 JavaScript 런타임입니다. 브라우저 밖에서 JavaScript를 실행할 수 있게 해주며, 비동기 I/O와 이벤트 기반 아키텍처로 높은 동시성을 제공합니다.

이벤트 루프 시각화

Node.js의 핵심인 이벤트 루프를 3D로 관찰하세요. 다양한 타입의 태스크가 이벤트 루프를 통해 처리되는 과정을 볼 수 있습니다.

⟳ 버튼을 눌러 이벤트 루프에 태스크를 추가하세요

이벤트 루프 단계

Timers → Pending I/O → Poll → Check → Close

⏱ Timers 단계

setTimeout()과 setInterval()의 콜백이 실행됩니다. 최소 지연 시간이 지난 콜백부터 순서대로 실행합니다.

💾 Poll 단계

I/O 관련 콜백을 실행합니다. 파일 읽기, 네트워크 요청 완료 등의 콜백이 여기서 처리됩니다.

⚡ Microtask Queue

Promise의 .then()과 process.nextTick()은 매 단계 사이에서 먼저 처리됩니다. 가장 높은 우선순위를 가집니다.

Express 프레임워크

Express는 Node.js의 가장 널리 쓰이는 웹 프레임워크로, 미들웨어 기반의 깔끔한 구조를 제공합니다.

const express = require('express'); const app = express(); // 미들웨어 app.use(express.json()); // 라우트 app.get('/api/users', (req, res) => { res.json({ users: [{ id: 1, name: '김철수' }] }); }); app.post('/api/users', (req, res) => { const { name, email } = req.body; res.status(201).json({ id: Date.now(), name, email }); }); app.listen(3000, () => { console.log('Express server on port 3000'); });

비동기 패턴

// 1. 콜백 (Callback Hell) fs.readFile('a.txt', (err, a) => { fs.readFile('b.txt', (err, b) => { fs.readFile('c.txt', (err, c) => { console.log(a + b + c); // 중첩 깊어짐 }); }); }); // 2. Promise 체이닝 readFileAsync('a.txt') .then(a => readFileAsync('b.txt')) .then(b => readFileAsync('c.txt')) .then(c => console.log(c)); // 3. async/await (권장) async function readAll() { const a = await readFileAsync('a.txt'); const b = await readFileAsync('b.txt'); const c = await readFileAsync('c.txt'); console.log(a + b + c); }

한 줄로 이해하기

Node.js는 "브라우저 밖에서 자바스크립트를 실행하는 런타임"이고, Express는 그 위에서 웹 서버를 쉽게 만들게 해주는 가장 인기 있는 프레임워크입니다.

왜 중요한가?

슬랙, 넷플릭스 일부, 우버, 페이팔, LinkedIn 등 거대 서비스 백엔드가 Node.js 위에서 돕니다. 프론트엔드 개발자가 같은 언어로 백엔드까지 만들 수 있어 풀스택 진입 장벽이 낮고, 스타트업에서 가장 빠르게 시작할 수 있는 스택 중 하나입니다.

🌱 핵심 용어

  • 런타임: 자바스크립트를 실제로 실행하는 환경(V8 엔진)
  • 이벤트 루프: 비동기 작업을 차례로 처리하는 핵심 메커니즘
  • npm/pnpm: 패키지 매니저 — 의존성 설치 도구
  • 미들웨어: 요청-응답 사이에서 실행되는 함수
  • 라우터: URL 패턴을 처리 함수에 연결

핵심 개념 풀어쓰기

Node.js의 핵심은 "비동기·논블로킹 I/O"입니다. 파일을 읽거나 DB를 조회하는 동안 다른 일을 합니다. 식당 점원이 한 손님 주문을 받고 주방에 넘긴 뒤 바로 다음 손님 주문을 받는 것과 같습니다. Express는 이 위에 라우팅·미들웨어 같은 친숙한 추상을 제공해 5분 안에 REST API를 만들 수 있게 합니다.

💡 학습 TIP: 처음에는 `npx create-express-app` 대신 빈 폴더에서 `npm init -y` → `npm i express` → app.js를 직접 작성해보세요. 어떤 줄이 무엇을 하는지가 또렷해집니다.

한 페이지 요약

  • Node.js = V8 + 이벤트 루프 + libuv
  • Express = 라우팅 + 미들웨어 프레임워크
  • 비동기 I/O로 동시 처리량을 확보
  • npm 생태계가 압도적으로 풍부
  • 가벼우면서 풀스택 진입에 유리
중 · 응용

이론 깊이 들어가기

Node.js의 이벤트 루프는 6단계 페이즈(timers → pending callbacks → idle/prepare → poll → check → close)로 구성됩니다. setImmediate vs setTimeout, process.nextTick, Promise(microtask)의 우선순위 차이가 여기서 결정됩니다. libuv 워커 스레드 풀은 기본 4개로 fs, crypto, dns 같은 시스템 호출을 처리합니다. CPU 집약 작업은 worker_threads로 분리하지 않으면 이벤트 루프가 막혀 전체 응답이 느려집니다. Express의 미들웨어는 함수 시그니처 (req, res, next)를 따르며, next() 호출 여부와 에러 미들웨어(err, req, res, next)의 4-인자 시그니처가 분기 기준입니다.

핵심 코드 패턴

const express = require('express');
const app = express();
app.use(express.json());
app.get('/api/users/:id', async (req, res, next) => {
  try {
    const user = await db.users.findById(req.params.id);
    if (!user) return res.status(404).json({error:'not found'});
    res.json(user);
  } catch (e) { next(e); }
});

대표 예제

📐 예제 1: 라우터 분리

routes/users.js에서 `const router = express.Router(); router.get('/', ...)` 후 `app.use('/api/users', router)`로 모듈화합니다. 파일이 커지면 도메인 단위로 분리합니다.

📐 예제 2: 에러 처리 미들웨어

모든 라우트 뒤에 (err, req, res, next) 시그니처의 미들웨어를 두어 비동기 에러를 한 곳에서 잡고, 환경별로 스택을 노출할지 결정합니다.

응용 사례

SaaS의 백오피스 API는 Express + TypeScript + Prisma 조합이 흔합니다. 인증은 Passport.js 또는 직접 작성한 JWT 미들웨어, 검증은 zod/joi, 문서는 swagger-jsdoc으로 자동 생성하는 식으로 결합합니다. Express 5.x부터는 비동기 핸들러의 에러가 자동으로 next로 전파됩니다.

⚠️ 자주 하는 실수: bcrypt.hashSync 같은 동기 호출을 요청 핸들러에서 그대로 쓰면 이벤트 루프가 막힙니다. 비동기 버전을 쓰거나 워커 스레드로 분리하세요.
상 · 심화

전문가 레벨 이해

대규모 Node.js 시스템에서는 메모리·이벤트 루프·GC 관측이 핵심입니다. clinic.js, 0x, autocannon으로 핫스팟을 찾고, --prof와 --inspect로 V8 프로파일을 분석합니다. 이벤트 루프 지연(event loop lag)을 ms 단위로 모니터링하고 50ms 이상 지속되면 알람을 울리는 것이 SRE 베스트 프랙티스입니다. Express는 단순하지만 한계가 있어 Fastify, NestJS, Hono로 옮기는 사례가 늘고 있습니다. Fastify는 schema-first JSON 직렬화로 Express보다 2~3배 빠른 처리량을 보입니다. Nest는 DI·모듈·데코레이터 기반으로 대규모 협업에 유리하지만 학습 곡선이 있습니다.

최신 연구·동향

Node.js 20+ 는 빌트인 test runner, fetch 글로벌, single-executable application(SEA)을 도입했습니다. Bun과 Deno가 대안 런타임으로 부상하며, Bun은 Zig 기반으로 시작 시간과 처리량에서 우위를 보이고 Deno는 보안 모델과 표준 라이브러리에 강점이 있습니다. Cloudflare Workers, Vercel Edge처럼 V8 isolate를 직접 쓰는 엣지 런타임은 일부 Node API만 지원하므로 코드 호환성을 의식해 작성해야 합니다.
