Chapter 1 — Server Fundamentals & Client-Server Model

클라이언트-서버 통신 시뮬레이터 포함

클라이언트-서버 모델

모든 웹 서비스의 근간은 클라이언트-서버 모델입니다. 브라우저(클라이언트)가 서버에 요청을 보내고, 서버가 처리 후 응답을 돌려보냅니다. 이 단순한 원리가 인터넷의 모든 서비스를 작동시킵니다.

🖥️ 서버(Server)란?

클라이언트의 요청을 받아 처리하고 응답을 돌려주는 프로그램입니다. 하드웨어가 아니라 소프트웨어 개념이며, 여러분의 노트북에서도 서버를 실행할 수 있습니다.

서버-클라이언트 통신 시뮬레이터

아래에서 "요청 보내기" 버튼을 눌러 클라이언트에서 서버로 데이터가 전달되고 응답이 돌아오는 과정을 관찰하세요.

📡 버튼을 눌러 클라이언트→서버→클라이언트 통신을 시뮬레이션하세요

TCP/IP 기초

인터넷 통신의 기반이 되는 TCP/IP 프로토콜은 데이터를 작은 패킷으로 나누어 전송합니다. TCP는 신뢰성 있는 전송을 보장하고, IP는 패킷의 주소 지정과 라우팅을 담당합니다.

🤝 TCP 3-Way Handshake

연결 수립 과정: ① 클라이언트가 SYN 전송 → ② 서버가 SYN-ACK 응답 → ③ 클라이언트가 ACK 전송. 이 세 단계를 거쳐야 데이터 전송이 시작됩니다.

🔌 포트 번호 체계

한 서버에서 여러 서비스를 구분하는 논리적 번호입니다. HTTP는 80, HTTPS는 443, 개발 서버는 주로 3000, 5000 등을 사용합니다. 0~1023은 잘 알려진 포트, 1024~49151은 등록된 포트입니다.

첫 번째 서버 만들기

Node.js로 가장 간단한 HTTP 서버를 만들어봅시다:

const http = require('http'); const server = http.createServer((req, res) => { console.log(`${req.method} ${req.url}`); res.writeHead(200, { 'Content-Type': 'text/plain' }); res.end('Hello, Backend World!'); }); server.listen(3000, () => { console.log('Server listening on port 3000'); });
node server.js Server listening on port 3000 → GET / 200 → GET /favicon.ico 200

동시성 모델

서버는 동시에 수많은 요청을 처리해야 합니다. 대표적인 세 가지 동시성 모델이 있습니다:

🔄 멀티 프로세스 (Apache prefork)

요청마다 새 프로세스를 생성합니다. 안정적이지만 메모리 비용이 큽니다. 각 프로세스가 독립적이므로 하나가 죽어도 전체에 영향이 적습니다.

🧵 멀티 스레드 (Tomcat)

프로세스 내에서 여러 스레드를 사용합니다. 메모리 효율이 좋지만 공유 자원 동기화(lock)가 필요합니다.

♻️ 이벤트 루프 (Node.js, Nginx)

단일 스레드로 수만 개의 연결을 비동기적으로 처리합니다. I/O 작업이 많은 서비스에 특히 효율적입니다. 챕터 3에서 자세히 다룹니다.

💡 탐구 과제

위 시뮬레이터에서 동시 요청 수를 늘려보세요. 요청이 많아질수록 서버 처리 과정이 어떻게 변하는지 관찰해보세요.

하 · 입문

한 줄로 이해하기

서버는 클라이언트(브라우저, 앱)의 요청을 받아 처리하고 결과를 돌려주는 "항상 켜져 있는 프로그램"입니다.

왜 중요한가?

인스타그램에 사진을 올리면 서버가 받아 저장하고 친구들에게 보여줍니다. 카카오톡 메시지, 쿠팡 검색, 유튜브 영상 재생 모두 서버를 거칩니다. 우리가 매일 쓰는 거의 모든 서비스가 서버 위에서 돌아가므로, 서버의 동작 원리를 이해하는 것은 백엔드 개발의 출발점입니다.

🌱 핵심 용어

  • 클라이언트: 요청을 보내는 쪽 — 브라우저, 모바일 앱
  • 서버: 요청을 받아 처리하는 프로그램
  • 요청(Request): 클라이언트가 보내는 작업 의뢰
  • 응답(Response): 서버가 돌려주는 처리 결과
  • 포트(Port): 한 컴퓨터의 여러 서비스를 구분하는 번호

핵심 개념 풀어쓰기

서버를 식당에 비유해보세요. 손님(클라이언트)이 주문(요청)을 하면 점원과 주방(서버)이 음식을 만들어 가져다 줍니다(응답). 한 명만 오는 게 아니라 동시에 여러 명이 주문하므로, 서버는 동시 요청을 효율적으로 처리할 줄 알아야 합니다. 손님이 너무 많으면 대기 줄이 길어지듯, 서버 처리 능력이 부족하면 응답이 느려집니다. 그래서 서버를 여러 대로 늘리거나(스케일 아웃) 한 대의 성능을 높이는(스케일 업) 방법을 씁니다.

💡 학습 TIP: 처음에는 "내 컴퓨터에 작은 서버 띄워보기"부터 시작하세요. Node.js나 Python 한 줄로 가능합니다.

한 페이지 요약

  • 서버는 요청을 받아 응답을 돌려주는 프로그램
  • 클라이언트와 서버는 네트워크로 연결
  • 포트로 같은 컴퓨터의 여러 서비스 구분
  • 요청-응답이 백엔드의 가장 기본 흐름
  • 스케일 업/아웃으로 처리 능력 확장
중 · 응용

이론 깊이 들어가기

서버는 TCP/IP 4계층 모델 기준으로 응용 계층(HTTP)에서 요청을 처리합니다. OS는 listen 시스템 콜로 포트를 열어 두고 accept로 연결을 받아들입니다. 동시에 여러 클라이언트를 처리하기 위해 동시성 모델이 필요합니다. 멀티 프로세스(Apache prefork), 멀티 스레드(Tomcat), 이벤트 루프(Node.js, Nginx) 세 가지가 대표적입니다. 멀티 프로세스는 안정적이지만 메모리 비용이 큽니다. 멀티 스레드는 효율적이지만 동기화 이슈가 있습니다. 이벤트 루프는 단일 스레드로 수만 연결을 비동기 처리해 I/O 위주 서비스에 유리합니다. 실무에서는 Nginx 같은 리버스 프록시 뒤에 애플리케이션 서버를 두는 구조가 일반적입니다.

핵심 코드 패턴

// Node.js 기본 HTTP 서버
const http = require('http');
http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type':'application/json'});
  res.end(JSON.stringify({ok:true}));
}).listen(3000);

대표 예제

💻 예제 1: 동시 접속 처리

이벤트 루프 서버는 한 요청이 DB 응답을 기다리는 동안 다른 요청을 받아 처리합니다. 콜백 → Promise → async/await 순으로 비동기를 표현합니다.

💻 예제 2: 리버스 프록시 구성

Nginx가 80/443에서 외부 요청을 받아 내부 3000번 Node 앱으로 전달합니다. SSL 종단, 정적 파일 서빙, gzip 압축을 프록시 계층에서 분리합니다.

응용 사례

대형 커머스는 트래픽 폭증 시간을 대비해 오토스케일링을 씁니다. CPU 70%를 넘으면 인스턴스를 자동으로 추가하고 트래픽이 줄면 줄입니다. 글로벌 서비스는 지역별 엣지 로케이션에 서버를 분산해 응답 속도를 줄입니다.

⚠️ 자주 하는 실수: 이벤트 루프에서 동기 블로킹 코드를 호출해 전체 서버가 멈추는 경우가 흔합니다. CPU 집약 작업은 워커 스레드로 분리하세요.
상 · 심화

전문가 레벨 이해

대규모 분산 서버에서는 단일 노드 성능보다 노드 간 일관성·가용성이 더 큰 도전입니다. CAP 정리에 따라 네트워크 분할 시 일관성과 가용성 중 하나만 선택해야 합니다. 결제 시스템은 강한 일관성을, 피드 시스템은 최종 일관성으로 분리 설계합니다. C10K, C10M 문제는 epoll, kqueue, io_uring 같은 OS 수준 비동기 I/O로 풉니다. TCP backlog, SO_REUSEPORT, TCP_NODELAY 같은 소켓 옵션 튜닝이 처리량에 직접적 영향을 줍니다. 컨센서스 알고리즘(Raft, Paxos)으로 리더 선출과 상태 복제를 안전하게 수행하고, 벡터 클럭이나 CRDT로 분산 상태 충돌을 해소합니다.

최신 연구·동향

Kubernetes가 컨테이너 오케스트레이션 표준이 되었고, eBPF는 커널에서 안전하게 코드를 실행해 Cilium 같은 사이드카리스 L7 네트워킹을 가능하게 했습니다. 서버리스(AWS Lambda, Cloudflare Workers)는 V8 isolate를 활용해 ms 단위 콜드 스타트를 달성합니다. 엣지 컴퓨팅은 사용자 가장 가까운 PoP에서 동적 코드를 실행해 글로벌 레이턴시를 줄입니다.
