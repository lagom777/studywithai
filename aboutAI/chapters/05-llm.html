7. 대규모 언어모델 (Large Language Models)

🧒 LLM (ChatGPT) 작동 원리

인터넷 전부 읽고 "다음 단어 맞추기" 게임을 수조 번 연습한 모델. 그러다 보니 번역·요약·코딩·수학까지 다 하게 됨.

  • 사전학습 = 태어나서 책만 10억 권 읽은 도서관 귀신
  • RLHF 정렬 = 예절 교육 ("험한 말 쓰지 마", "도움되게 말해")
  • RAG = 책상에 사전 펼쳐두고 참고하며 답하기
  • Chain-of-Thought = "단계별로 생각해봐"라고 하면 훨씬 잘함
스케일링 법칙: 모델 크기를 2배 → 4배 → 8배로 키우면 성능도 꾸준히 좋아진다. 어느 크기 넘으면 갑자기 새로운 능력이 생김(창발). 작은 모델은 3자리 덧셈 못 하다가 어느 순간 됨.
o1 / DeepSeek-R1: "생각할 시간을 더 줄게" — 답하기 전에 오래 고민하는 모델. 수학·코딩에서 도약적 성능.

7.1 사전학습의 본질: 다음 토큰 예측

LLM의 사전학습 목표는 놀라울 정도로 단순하다: \[ \mathcal{L}_{\text{LM}}(\theta) = -\sum_{t=1}^T \log p_\theta(x_t \mid x_{자기회귀 언어모델링이며, 토크나이저(주로 BPE 또는 SentencePiece)로 분할된 토큰열에 대해 수행된다. Sutskever(2023)의 주장: 다음 토큰을 정확히 예측하기 위해서는 세계에 대한 모델이 필요하므로, 이 단순한 목표가 압축을 통해 지능의 씨앗을 발아시킨다.

토큰화 (Tokenization)

Byte-Pair Encoding (BPE)은 가장 빈번한 바이트 쌍을 반복적으로 병합하여 서브워드 어휘를 구축한다. 현대 모델은 50K~200K 규모의 어휘를 사용한다. 토큰화의 선택은 다국어 성능, 숫자 처리, 코드 성능에 큰 영향을 미친다.

7.2 스케일링 법칙 (Scaling Laws)

Kaplan et al.(2020)은 LLM 손실이 파라미터 수 $N$, 데이터 크기 $D$, 계산량 $C$에 대해 거듭제곱 관계를 따름을 보였다: \[ \mathcal{L}(N) = (N_c / N)^{\alpha_N}, \quad \alpha_N \approx 0.076 \] Hoffmann et al.(2022)의 Chinchilla 연구는 Kaplan의 처방을 수정하여, 주어진 계산 예산 하에서 최적 모델은 파라미터와 토큰을 대략 동등한 비율로 확장해야 함을 보였다: $N \propto C^{0.5}$, $D \propto C^{0.5}$. 경험칙으로 "토큰 수 $\approx$ 20 × 파라미터 수"가 Chinchilla-optimal로 알려졌다. 최근(Llama 3, 2024)에는 추론 비용을 고려해 더 많은 데이터로 작은 모델을 과훈련하는 경향이 있다.

그림 9. 스케일링 법칙. 연산량(FLOPs)이 증가할수록 손실이 거듭제곱 법칙을 따라 감소한다. 로그-로그 좌표에서 직선을 그린다.

7.3 창발적 능력 (Emergent Abilities)

Wei et al.(2022)은 특정 능력(다단계 산술, 다국어 추론 등)이 특정 스케일 이하에서는 무작위 수준이다가 임계 스케일을 넘어서면 급격히 출현함을 보고하였다. 이를 창발이라 부른다. Schaeffer et al.(2023)은 이것이 실제 불연속적 상전이가 아니라 평가 지표의 불연속성에 기인한 인위적 현상이라 주장하여 논쟁 중이다. 그럼에도 in-context learning, chain-of-thought, 코드 생성 등은 규모 의존적으로 질적 차이를 보인다는 점은 경험적으로 명백하다.

7.4 정렬과 RLHF

사전학습된 기본 모델(base model)은 도움됨이나 무해성이 아니라 단지 그럴듯함을 학습할 뿐이다. 실용적 어시스턴트로 만드는 과정이 정렬(alignment)이다. Ouyang et al.(2022) InstructGPT의 파이프라인:

  1. 지도 미세조정(SFT): 사람이 작성한 시연 데이터로 파인튜닝.
  2. 보상 모델(RM) 학습: 쌍별 선호 데이터 $(y_w \succ y_l \mid x)$로 보상함수 학습. Bradley-Terry 모델에 기반한 손실: \[ \mathcal{L}_{\text{RM}} = -\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l)) \]
  3. PPO를 통한 강화학습: 보상 $r_\phi$를 최대화하되, 기준 모델과의 KL 발산으로 정규화: \[ \max_\theta \mathbb{E}[r_\phi(x,y) - \beta \cdot D_{\text{KL}}(\pi_\theta \| \pi_{\text{ref}})] \]

Rafailov et al.(2023)의 DPO (Direct Preference Optimization)는 명시적 보상 모델과 RL 단계를 우회하여, 다음의 닫힌 형태의 손실로 직접 최적화한다: \[ \mathcal{L}_{\text{DPO}} = -\mathbb{E}\!\left[\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right] \] 단순성과 안정성 때문에 현재 오픈소스 정렬의 표준에 가깝다. 후속 기법: IPO, KTO, GRPO, SimPO 등.

그림 10. RLHF (Reinforcement Learning from Human Feedback) 3단계 파이프라인. InstructGPT/ChatGPT가 개척한 현대 정렬 표준.

헌법 AI (Constitutional AI, Anthropic)

Bai et al.(2022)의 접근은 사람 레이블러 대신 AI 피드백(RLAIF)을 사용하며, 모델이 "헌법"(일련의 원칙)을 참조해 자기비판과 수정을 수행하도록 한다. 이는 인간 라벨링의 병목과 일관성 문제를 완화한다.

7.5 추론 능력과 테스트-타임 스케일링

Wei et al.(2022) Chain-of-Thought (CoT) prompting: "step by step"이라는 단순한 지시만으로 다단계 추론 문제 성능이 극적으로 향상됨을 보임. 이는 LLM이 내부적으로 문제 해결 궤적을 생성할 수 있음을 시사한다.

2024년 OpenAI o1의 등장은 새로운 패러다임을 열었다: 테스트-타임 계산 스케일링. 사전학습 계산을 늘리는 대신, 추론 시 긴 내부 사고 사슬을 생성하도록 학습된 모델은 수학·코딩·과학 벤치마크에서 도약적 성능을 보였다. DeepSeek-R1(2025)은 순수 RL(GRPO)만으로 추론 능력이 창발함을 공개적으로 입증하였다.

검색 증강 생성 (RAG)

Lewis et al.(2020)은 질의와 관련된 문서를 외부 지식베이스에서 검색하여 프롬프트에 주입함으로써, 파라미터에 지식을 저장하는 부담을 줄이고 최신성·출처 제공이 가능한 프레임워크를 제시했다. RAG의 구성요소: 임베딩 모델, 벡터 데이터베이스(FAISS, Milvus), 재순위(reranker), 생성기.

한 줄로 이해하기

대규모 언어모델(LLM)은 인터넷 규모의 텍스트를 읽고 "다음 단어 맞히기" 게임을 수십억 번 반복해서 언어와 세상을 익힌 거대한 신경망이에요. ChatGPT, Claude, Gemini가 모두 LLM이고, 이름의 "Large"는 매개변수가 보통 수십억 ~ 수조 개라는 뜻이에요.

왜 중요한가?

① 학생이 보고서 초안을 쓸 때, 회사원이 이메일을 작성할 때 LLM은 시간을 절약해줍니다. ② 프로그래머는 GitHub Copilot, Cursor 같은 도구로 코드를 자동완성받아 생산성이 2~3배 올라가요. ③ 의사·법조인은 방대한 문서를 LLM에게 요약시키고 사례를 검색하는 데 활용합니다. ④ 외국어 학습자는 LLM과 자연스러운 대화를 통해 회화 연습을 합니다. 한마디로 LLM은 21세기의 "사고 보조 도구"예요.

🌱 핵심 용어

  • 토큰: 모델이 처리하는 텍스트 조각 (단어 또는 단어의 일부)
  • 매개변수: 모델이 학습한 "기억"의 단위 (예: GPT-3은 175B개)
  • 사전학습(pretraining): 인터넷 텍스트로 다음 토큰 맞히기 훈련
  • 미세조정(fine-tuning): 특정 작업·스타일에 맞춰 추가 학습
  • RLHF: 사람의 피드백으로 모델을 정렬하는 강화학습
  • 프롬프트: 모델에게 주는 입력 지시문
  • 할루시네이션: 그럴듯하지만 사실이 아닌 답을 만들어내는 현상

핵심 개념 풀어쓰기

LLM이 "다음 단어 맞히기"를 어떻게 배우는지 비유로 설명해볼게요. 어린이에게 빈칸이 뚫린 책 1조 권을 주고 "여기 들어갈 단어가 뭘까?"를 계속 물어봐요. 처음에는 아무 단어나 찍지만, 수억 번 반복하면 문맥에 어울리는 단어를 점점 잘 맞히게 됩니다. 이 과정에서 어린이는 문법, 상식, 역사, 과학, 코딩, 농담, 시 쓰기까지 자연스럽게 익히죠. 이게 LLM이에요. 다만 LLM은 "이해"하는 게 아니라 "통계적 패턴을 익힌" 것이라서, 가끔 그럴듯한 거짓말(할루시네이션)을 하기도 합니다. 그래서 사실 확인이 중요해요.

💡 학습 TIP: LLM을 "백과사전 + 비서 + 토론 상대"로 생각하세요. 단순 검색은 구글이 더 좋고, 복잡한 글쓰기·요약·번역·코딩은 LLM이 더 좋습니다. 그리고 중요한 답은 항상 출처 확인을 해야 한다는 점을 잊지 마세요.

한 페이지 요약

  • LLM = 거대한 Transformer + 인터넷 텍스트로 사전학습
  • 학습 목표는 단순히 "다음 토큰 예측"
  • 스케일이 커질수록 새로운 능력(추론, 코드, 번역)이 emergent하게 발현
  • RLHF·DPO로 사람 선호에 맞춰 정렬
  • 한계: 할루시네이션, 학습 데이터 시점 한계, 편향, 추론 비용

핵심 개념과 코드

# Hugging Face transformers로 LLM 사용
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

tok = AutoTokenizer.from_pretrained("meta-llama/Llama-3-8B-Instruct")
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3-8B-Instruct",
    torch_dtype=torch.bfloat16, device_map="auto"
)
prompt = "한국 수도는 어디인가? 한 문장으로."
inputs = tok(prompt, return_tensors="pt").to(model.device)
out = model.generate(**inputs, max_new_tokens=64,
                     temperature=0.7, top_p=0.9)
print(tok.decode(out[0], skip_special_tokens=True))

💻 예제 1 — In-Context Learning

모델 가중치를 업데이트하지 않고 프롬프트 안에 예시 몇 개만 보여주면 새 작업을 수행합니다. "프랑스 → Paris, 한국 → ?" 처럼요. GPT-3의 emergent 능력으로 발견되었고, induction head가 그 메커니즘으로 추정됩니다(Olsson 2022).

💻 예제 2 — Chain-of-Thought

"단계별로 생각해봐"를 프롬프트에 추가하면 산수·논리 문제 정답률이 급상승합니다. Wei(2022)는 8-shot CoT가 GSM8K에서 정확도를 18% → 57%로 끌어올림을 보였습니다. o1, R1 같은 reasoning 모델은 이를 자동화·내재화했습니다.

응용 사례

① RAG(Retrieval-Augmented Generation): 외부 문서를 벡터 검색으로 가져와 LLM이 참조하게 함. 사내 지식, 최신 뉴스 등 학습 데이터에 없는 정보 활용. ② 에이전트 시스템(AutoGPT, LangChain, Claude tool use): LLM이 도구(검색, 계산기, 코드 실행)를 호출해 복잡 작업 수행. ③ 코드 어시스턴트(Copilot, Cursor, Cline): IDE 내부에서 코드 생성·디버깅·리팩터링. ④ 의료/법률 특화 모델: Med-PaLM, Harvey AI는 도메인 특화 사전학습 + RLHF로 전문가 수준에 도달.

⚠️ 자주 하는 실수: LLM 출력을 사실로 그대로 신뢰하면 위험합니다. 특히 인용·통계·법률·의학 정보는 반드시 검증하세요. 또 prompt injection으로 외부 데이터의 악성 지시가 모델 행동을 바꿀 수 있으니, 시스템 프롬프트와 사용자 프롬프트의 분리, output sanitization이 필요합니다.
상 · 심화

전문가 레벨 이해

Scaling laws의 함수형은 L(N, D) = E + A/N^α + B/D^β로, 매개변수 N, 데이터 D에 대해 멱함수 감쇠를 보입니다. Chinchilla(Hoffmann 2022)는 compute-optimal frontier에서 N과 D가 거의 같은 비율로 증가해야 함을 도출했고, GPT-3(175B + 300B토큰)가 데이터 부족 상태였음을 보였습니다. 이후 Llama 2/3, Mistral, DeepSeek는 훨씬 많은 데이터(1T~15T)를 사용합니다. Emergent abilities(Wei 2022)는 일정 스케일을 넘어서야 능력이 갑자기 나타나는 현상으로, 일부는 평가 메트릭의 비연속성으로 인한 착시(Schaeffer 2023)이지만 in-context learning, multi-step reasoning은 진짜 emergent로 보입니다. RLHF의 한계는 RM의 over-optimization(Goodhart's law), reward hacking, mode collapse입니다. DPO는 closed-form 손실 −log σ(β log(π(y_w)/π_ref(y_w)) − β log(π(y_l)/π_ref(y_l)))로 단순화했지만 base model 품질에 더 민감합니다. Constitutional AI(Anthropic, Bai 2022)는 사람 라벨 대신 AI가 헌법 원칙으로 자기 비판·재작성하는 RLAIF를 제안했습니다. Inference 측면에서 KV cache, speculative decoding, FlashAttention, paged attention(vLLM), continuous batching이 throughput을 좌우합니다. Test-time scaling(o1, R1)은 inference compute를 늘려 reasoning 성능을 향상시키며, train-time scaling과 직교하는 새 차원입니다.

최신 연구와 frontier

① Reasoning 모델: OpenAI o1·o3, DeepSeek-R1, Claude 3.5 Sonnet의 extended thinking은 RL로 chain-of-thought를 직접 학습하여 수학·코딩에서 인간 전문가 수준에 근접. ② MoE 스케일링: DeepSeek-V3(671B 총/37B 활성), Mixtral, GLaM이 동일 inference 비용에서 더 큰 capacity. ③ Long context + retrieval: Gemini 2M context, NIAH(needle in a haystack) 평가, RAG의 진화. ④ Mechanistic interpretability: Anthropic의 Sparse Autoencoder로 16M개 features 추출, Golden Gate Claude 같은 feature steering 데모.

관련 분야 연결

  • 강화학습: PPO, REINFORCE, GRPO(R1) 알고리즘과 reward shaping
  • 분산 시스템: 3D parallelism (data, tensor, pipeline), ZeRO, FSDP
  • 인지과학: dual-process theory(System 1/2)와 fast/slow thinking 모델
  • 경제학·게임이론: alignment를 principal-agent problem으로 분석
  • 철학: Searle 중국어방, 의미·이해의 본질, AI 의식 논쟁
