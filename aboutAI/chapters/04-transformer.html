6. Attention과 Transformer

🧒 Transformer의 심장: 어텐션

어텐션 = 문장 읽을 때 중요한 단어에 형광펜 치기.

"The cat sat on the mat"에서 "sat(앉았다)"을 이해하려면 "cat(누가?)"을 봐야 한다. 어텐션이 "cat에 70% 주목, 나머지 30%"로 가중치를 매긴다.
Q·K·V는 도서관 비유:
  • Q (쿼리) = 내가 찾고 싶은 것 ("앉은 건 누구?")
  • K (키) = 각 책의 제목 명찰 ("나는 고양이야")
  • V (값) = 책 안의 실제 내용
  • Q와 K 궁합 맞추기 → 잘 맞는 책의 V를 많이 참고

💡 Transformer가 대단한 이유: 문장의 첫 단어와 마지막 단어가 얼마나 떨어져 있든 한 번에 관계를 볼 수 있다. RNN은 순서대로 읽어야 했는데, 이건 병렬로 본다.

Transformer 아키텍처(Vaswani et al., 2017)는 RNN의 순차성과 CNN의 지역성을 모두 버리고, 자기주의(self-attention)만으로 시퀀스를 처리한다. 현대 AI의 중심축이므로 상세히 다룬다.

그림 6. 스케일드 닷-프로덕트 어텐션의 계산 흐름. Q·Kᵀ로 유사도를 측정 → softmax로 정규화 → V에 가중합 적용. 이것이 Transformer의 심장이다.

6.1 스케일드 닷-프로덕트 어텐션

입력 시퀀스 $\mathbf{X} \in \mathbb{R}^{n\times d}$에 대해 세 가지 선형 사영을 만든다: \[ \mathbf{Q} = \mathbf{X}\mathbf{W}^Q, \quad \mathbf{K} = \mathbf{X}\mathbf{W}^K, \quad \mathbf{V} = \mathbf{X}\mathbf{W}^V \] 어텐션 출력은: \[ \mathrm{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \mathrm{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V} \tag{6.1} \] $\sqrt{d_k}$로의 스케일링은 softmax 포화를 방지한다. 어텐션의 각 출력은 모든 위치의 값의 가중합이므로, 시퀀스 내 임의 거리의 의존성을 단일 층에서 모델링할 수 있다(경로 길이 $O(1)$).

6.2 다중 헤드 어텐션 (Multi-Head Attention)

단일 어텐션 대신 $h$개의 병렬 헤드를 사용한다: \[ \mathrm{MultiHead}(\mathbf{X}) = \mathrm{Concat}(\mathrm{head}_1, \dots, \mathrm{head}_h)\mathbf{W}^O \] 각 헤드는 서로 다른 부분공간에서 관계를 포착하며, 이는 문법·의미·참조 등 다양한 언어적 관계에 대응함이 경험적으로 관찰된다.

6.3 Transformer 블록의 전체 구조

각 블록은 다음을 수행한다: 

x = x + MultiHeadAttention(LayerNorm(x))   # Pre-LN 방식 (현대 표준)
x = x + FFN(LayerNorm(x))
# FFN(x) = W_2 · σ(W_1 · x)  (보통 SwiGLU)

Pre-LayerNorm(Xiong et al., 2020)은 훈련 안정성을 크게 향상시켜 GPT-2 이후 표준이 되었다. 잔차 연결은 그래디언트 고속도로 역할을 한다.

그림 8. Transformer 블록의 해부도. 어텐션과 FFN이 교대로 쌓이며, 각각 잔차 연결과 LayerNorm으로 둘러싸인다. 이 블록이 수십~수백 번 반복되어 LLM이 된다.

6.4 위치 인코딩 (Positional Encoding)

자기주의는 순열 등변(permutation-equivariant)이므로 순서를 명시적으로 주입해야 한다.

  • 절대 위치: sinusoidal (Vaswani et al., 2017) 또는 학습된 임베딩.
  • 상대 위치: Shaw et al.(2018), T5-style.
  • RoPE (Rotary Position Embedding, Su et al., 2021): 쿼리·키 벡터를 회전시켜 상대 위치를 주입. 현재 Llama, Qwen, GPT-NeoX 등 대부분의 현대 LLM에서 채택.
  • ALiBi(Press et al., 2022): 어텐션 점수에 거리 비례 편향을 가산, 길이 외삽에 유리.

6.5 어텐션의 계산 복잡도와 해결책

표준 어텐션은 시퀀스 길이 $n$에 대해 $O(n^2 d)$의 시간·메모리를 요구하여, 긴 맥락 처리의 병목이다.

  • FlashAttention(Dao et al., 2022): 커널 융합과 타일링으로 메모리 I/O를 최소화. 근사가 아닌 정확한 계산.
  • 희소 어텐션: Longformer, BigBird 등 $O(n)$ 근사.
  • Mamba / SSM(Gu & Dao, 2023): 상태 공간 모델로 선형 복잡도 달성, 긴 맥락 후보.
  • Grouped-Query Attention (GQA)(Ainslie et al., 2023): 여러 쿼리 헤드가 하나의 키·값 헤드 공유. 추론 시 KV 캐시 감소.

6.6 인코더-디코더 변형

원본 Transformer는 번역을 위한 인코더-디코더였으나, 이후 세 계통으로 분화되었다:

  • 인코더 전용: BERT, RoBERTa — MLM으로 양방향 표현 학습. 분류·검색.
  • 디코더 전용: GPT 계열 — 인과 마스킹으로 자기회귀 생성. 현재 LLM의 지배적 아키텍처.
  • 인코더-디코더: T5, BART — 번역·요약 등 seq2seq.

한 줄로 이해하기

Transformer는 "문장의 모든 단어가 서로를 바라보면서" 어떤 단어가 어떤 단어와 관련이 있는지를 동시에 계산하는 신경망이에요. 2017년 구글의 "Attention is All You Need" 논문에서 처음 등장했고, 지금 GPT, Claude, Gemini 등 모든 거대 AI의 뼈대가 됐습니다.

왜 중요한가?

① ChatGPT와 대화할 때 앞에서 한 말을 기억하는 건 Transformer의 self-attention이 모든 토큰을 동시에 보기 때문이에요. ② 구글 번역이 자연스럽게 한국어 ↔ 영어를 번역하는 것도 2017년 이후 Transformer 기반으로 바뀐 결과입니다. ③ DALL·E나 Midjourney가 그림을 그리는 것도 Transformer 위에 디퓨전 모델을 얹은 구조예요. 한마디로 현대 AI의 모든 신기한 능력은 Transformer가 출발점입니다.

🌱 핵심 용어

  • Attention: 어떤 단어에 얼마나 집중할지 결정하는 메커니즘
  • Self-Attention: 같은 문장 안의 단어들끼리 서로 주목
  • Q, K, V: 질의(Query), 열쇠(Key), 값(Value) — 도서관 검색 비유로 설명
  • 토큰: 단어 또는 단어의 일부 (모델이 한 번에 처리하는 최소 단위)
  • Positional Encoding: 단어 순서 정보를 더해주는 신호
  • Encoder/Decoder: 입력을 이해하는 부분 / 출력을 생성하는 부분

핵심 개념 풀어쓰기

Self-attention을 도서관에 비유해볼게요. 여러분이 "사과의 색깔이 뭐지?"라는 질문(Query)을 들고 도서관에 갔어요. 모든 책에는 제목 카드(Key)와 내용(Value)이 있죠. 여러분의 질문과 가장 비슷한 제목 카드를 가진 책을 찾아 그 내용을 읽으면 됩니다. Transformer는 문장 속 모든 단어가 이런 식으로 서로의 책을 동시에 검색해요. 예를 들어 "그녀가 사과를 들고 빨갛다고 말했다"에서 "빨갛다"는 "사과"를 강하게 참조하고 "그녀"는 약하게 참조하는 식이죠. 이걸 한 번에 계산하니까 RNN처럼 한 단어씩 차례차례 처리하는 것보다 훨씬 빠르고, 멀리 떨어진 단어 간의 관계도 잘 잡아냅니다.

💡 학습 TIP: "Attention이 어렵다"고 느낄 땐 직접 작은 행렬로 손으로 계산해보세요. 4개 단어 × 차원 4의 Q, K, V를 만들고 softmax(QKᵀ/√d)V를 종이에 적어보면, 신비롭게 느껴졌던 어텐션이 단순한 가중평균임이 보입니다.

한 페이지 요약

  • 2017년 Vaswani et al. "Attention is All You Need"가 시작
  • RNN의 순차 처리를 self-attention으로 대체 → 병렬화·long-range 의존성 해결
  • Multi-Head Attention으로 여러 관점의 관계를 동시에 학습
  • Encoder-Decoder, Encoder-only(BERT), Decoder-only(GPT) 세 변종
  • 현대의 LLM, ViT, 디퓨전 모델까지 모두 Transformer 기반
중 · 응용

이론 깊이 들어가기

Vaswani et al.(2017)의 핵심 통찰은 "RNN의 순환 구조 없이도 시퀀스 모델링이 가능하다"였습니다. Self-attention은 입력 시퀀스 X ∈ R^(n×d)에 대해 세 개의 선형 변환 Q = XW_Q, K = XW_K, V = XW_V를 만들고, Attention(Q, K, V) = softmax(QK^T/√d_k)V로 가중합을 계산합니다. √d_k 스케일링은 dot-product의 분산 폭주를 막아 softmax의 그래디언트 안정성을 확보합니다. Multi-Head Attention은 h개의 attention을 병렬로 수행한 뒤 concat하고 선형변환합니다. 각 head가 서로 다른 관계 패턴(문법, 의미, 공참조 등)을 학습하도록 유도하는 효과가 있습니다. Position 정보는 sin/cos 기반 절대 인코딩(원논문) 또는 RoPE(Rotary Position Embedding, Su et al., 2021), ALiBi 같은 상대 인코딩으로 주입합니다. Transformer 블록은 (1) Multi-Head Self-Attention, (2) Position-wise Feed-Forward Network(FFN), 그리고 각각의 Residual + LayerNorm으로 구성됩니다. FFN은 dim_model → 4·dim_model → dim_model로 확장-축소되며, 사실 매개변수의 약 2/3가 여기 있습니다.

핵심 개념과 코드

import torch, math
import torch.nn as nn
import torch.nn.functional as F

class SelfAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.h, self.d = n_heads, d_model // n_heads
        self.qkv = nn.Linear(d_model, 3*d_model)
        self.proj = nn.Linear(d_model, d_model)
    def forward(self, x):
        B, T, D = x.shape
        q, k, v = self.qkv(x).chunk(3, dim=-1)
        q = q.view(B, T, self.h, self.d).transpose(1,2)
        k = k.view(B, T, self.h, self.d).transpose(1,2)
        v = v.view(B, T, self.h, self.d).transpose(1,2)
        att = (q @ k.transpose(-2,-1)) / math.sqrt(self.d)
        att = F.softmax(att, dim=-1)
        out = (att @ v).transpose(1,2).contiguous().view(B, T, D)
        return self.proj(out)

대표 예제

💻 예제 1 — BERT vs GPT

BERT(2018)는 Encoder-only로 양방향 마스킹 언어 모델링(MLM)을 수행합니다. 문장 이해, 분류, NER에 강합니다. GPT(2018~)는 Decoder-only로 인과적 마스킹을 사용해 다음 토큰을 예측합니다. 생성 작업에 강하며, 현재의 ChatGPT, Claude는 모두 GPT 계열 구조입니다.

💻 예제 2 — Vision Transformer (ViT)

이미지를 16x16 패치로 잘라 토큰화하고, CLS 토큰과 함께 Transformer Encoder에 넣으면 분류기가 됩니다. JFT-300M 같은 대규모 데이터에서 학습할 때 CNN을 능가하며, 현재 비전 백본의 표준 중 하나입니다.

응용 사례

① 코드 생성(GitHub Copilot, Cursor)은 GPT 계열 디코더로 다음 토큰을 예측해 코드를 자동완성합니다. ② 단백질 구조 예측 AlphaFold 2의 Evoformer는 Transformer의 변종으로 MSA(Multiple Sequence Alignment)와 pair representation을 동시에 처리합니다. ③ 음성 합성(VALL-E, Bark)은 오디오 토큰화 후 Transformer로 생성합니다. ④ 추천 시스템에서도 SASRec, BERT4Rec 같이 사용자의 클릭 시퀀스를 Transformer로 모델링합니다.

⚠️ 자주 하는 실수: Self-attention의 시간·메모리 복잡도는 O(n²d)입니다. 시퀀스 길이가 8K, 32K가 되면 메모리가 폭증하므로 FlashAttention, sliding window, Mamba 같은 효율 기법이 필요합니다. 또 Layer Norm 위치(pre-LN vs post-LN)에 따라 학습 안정성이 크게 달라집니다.
상 · 심화

전문가 레벨 이해

Self-attention은 dynamic weight를 가진 그래프 신경망의 한 형태로 해석할 수 있습니다. 각 토큰이 노드이고, attention 가중치가 동적으로 결정되는 edge weight입니다. 이론적으로 Transformer는 Universal Approximator이며(Yun et al., 2020), 충분한 깊이와 폭이 주어지면 임의의 sequence-to-sequence 함수를 근사할 수 있습니다. Mechanistic interpretability 관점에서 Anthropic의 Transformer Circuits 연구(Elhage 2021, Olsson 2022)는 attention head를 induction head, copying head, name mover head 등으로 분해하고, in-context learning이 induction head에 의해 구현됨을 보였습니다. RoPE(Rotary Position Embedding)는 회전 행렬 R_(θm)을 Q, K에 곱하여 상대 위치 m-n을 내적에 자연스럽게 삽입합니다. 이는 long context 일반화의 핵심이 되었고, NTK-aware scaling, YaRN, Position Interpolation 같은 후속 기법으로 RoPE 기반 모델의 컨텍스트 확장이 가능해졌습니다. FlashAttention(Dao 2022, 2023)은 SRAM 친화적 타일링으로 attention의 IO 복잡도를 O(n²d/M)에서 O(n²d²/M)로 개선해 실효 속도를 2-4배 높였습니다. Sparse attention(Longformer, BigBird), linear attention(Performer, Linformer), 그리고 SSM(Mamba, S5)는 quadratic complexity를 우회하는 다른 접근들입니다.

수학적 기초

$$\text{Scaled Dot-Product Attention: } \text{Attn}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$ $$\text{Multi-Head: } \text{MHA}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O$$ $$\text{RoPE: } R_{\theta_m} q \cdot R_{\theta_n} k = q \cdot R_{\theta_{m-n}} k$$ $$\text{FFN: } \text{FFN}(x) = W_2 \cdot \text{GeLU}(W_1 x + b_1) + b_2$$

최신 연구와 frontier

① Long context: Gemini 2M, Claude 200K, GPT-4 128K 등이 RoPE 보간, Ring Attention, ALiBi로 컨텍스트를 확장 중입니다. ② Sub-quadratic 대안: Mamba, RWKV, RetNet, Hyena가 long-context에서 Transformer와 경쟁하며 일부 작업에서 더 우수합니다. ③ MoE Transformer: Mixtral, DeepSeek-MoE는 routing function으로 토큰을 expert에 분배해 활성 매개변수를 줄입니다. ④ Diffusion Transformer(DiT): Sora, Stable Diffusion 3는 U-Net 대신 Transformer를 디퓨전 백본으로 채택했습니다.

관련 분야 연결

  • 그래프 신경망: Transformer = fully-connected graph + dynamic attention
  • 커널 방법: Performer는 attention을 random feature kernel로 근사
  • 제어이론: SSM(Mamba)는 linear time-invariant system의 직접 근사
  • 정보이론: attention entropy 분석, head 중요도 측정
