7장. 분산 데이터베이스

1. 분산 DB란?

쉬운 설명

하나의 데이터베이스를 여러 컴퓨터에 나눠 저장하는 것입니다. 예를 들어, 당신이 운영하는 온라인 쇼핑몰이 매우 커져서 하나의 서버로는 감당할 수 없다면, 데이터를 여러 서버에 분산시킵니다.

장점: 한 서버가 망가져도 다른 서버가 서비스를 유지합니다. 동시에 더 많은 사람들을 처리할 수 있습니다.

비유: 하나의 도서관이 너무 커져서 여러 지점을 열어 책을 분산 보관하는 것과 같습니다.

학술 설명

분산 데이터베이스 시스템(DDBMS)은 물리적으로 떨어진 여러 노드에 데이터가 분산되어 저장되며, 네트워크를 통해 통신하는 시스템입니다.

확장성 전략:

  • 수평 확장(Scale-out): 더 많은 서버를 추가. 분산 DB의 목표.
  • 수직 확장(Scale-up): 기존 서버의 성능을 높임. 비용이 높음.

핵심 과제: 분산된 데이터의 일관성, 가용성, 신뢰성 유지

2. CAP 정리

쉬운 설명

분산 DB는 3가지 중 최대 2가지만 완벽히 보장할 수 있습니다:

  • 일관성(Consistency): 모든 노드가 같은 데이터를 본다
  • 가용성(Availability): 언제든 응답한다
  • 분할내성(Partition Tolerance): 네트워크 끊김을 견딘다

삼각형 비유: 삼각형 중에 2개의 꼭짓점만 잡을 수 있다면, 하나는 버려야 한다는 뜻입니다.

실제: 은행(일관성 중심) vs SNS(가용성 중심)

학술 설명

Brewer의 CAP 정리(2000): 분산 시스템은 C, A, P 중 최대 2개만 동시에 만족할 수 없다.

Gilbert & Lynch의 증명(2002): 네트워크 분할 상황에서 일관성과 가용성을 동시에 보장 불가능함을 증명.

실무 분류:

  • CA: 전통적 RDBMS (네트워크 분할 가정하지 않음)
  • AP: Dynamo, Cassandra (최종 일관성)
  • CP: MongoDB, HBase (가용성 포기)

PACELC 확장: 네트워크가 정상일 때 Latency-Consistency 트레이드오프도 고려

3. 샤딩(Sharding)

쉬운 설명

데이터를 규칙에 따라 여러 서버로 분산하는 방법입니다.

도서관 비유: 도서관 분관을 만들 때, A~M 성씨 책은 강남지점에, N~ㅎ 성씨 책은 강북지점에 보관합니다.

장점: 각 서버가 작은 데이터만 관리하므로 빠릅니다.

문제: 한 서버에 데이터가 몰릴 수 있습니다 (핫스팟).

학술 설명

샤딩 전략:

  • 해시 샤딩: hash(key) % N으로 샤드 결정. 균등 분산.
  • 레인지 샤딩: 범위로 분할. 범위 쿼리에 유리하지만 핫스팟 발생 가능.
  • 디렉토리 샤딩: 매핑 테이블 사용. 유연하지만 조회 비용 증가.

컨시스턴트 해싱: 노드 추가/제거 시 대부분의 데이터 이동을 피함.

핫스팟 문제: 일부 샤드에 데이터가 집중. 해결책: 샤드 재분배, 마이크로 샤딩

4. 복제(Replication)

쉬운 설명

같은 데이터를 여러 서버에 복사해 저장하는 방법입니다.

도서관 비유: 인기 있는 책은 여러 도서관에 비치합니다. 하나가 불타도 다른 곳에 책이 남아있습니다.

장점: 서버 장애에 강합니다. 읽기 성능도 향상됩니다.

문제: 모든 복제본을 최신으로 유지해야 합니다.

학술 설명

복제 토폴로지:

  • 마스터-슬레이브: 주 노드가 쓰기, 부 노드가 읽기. 단순하지만 주 노드가 병목.
  • 멀티마스터: 모든 노드가 쓰기 가능. 충돌 해결이 복잡.
  • 피어-투-피어: 모든 노드가 동등. 가장 탄력적.

일관성 모델:

  • 강한 일관성: 모든 읽기가 최신 데이터 보장. 느림.
  • 최종 일관성(Eventual Consistency): 결국 일관성 도달. 빠르지만 임시 불일치 발생.

쿼럼(Quorum) 기반 복제: N개 복제본 중 W개 쓰기, R개 읽기로 일관성 조정
