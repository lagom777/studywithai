1장. 관계형 데이터베이스 기초

1.1 관계형 모델이란?

🎯 쉽게 이해하기

관계형 데이터베이스는 데이터를 표(테이블)로 관리하는 방식입니다. 마치 엑셀 스프레드시트처럼 행과 열로 구성되어 있어요.

예시: 학교의 학생 명부

학번 이름 학년 전공
001 김철수 2학년 컴퓨터공학
002 이영희 3학년 경영학

이런 식으로 데이터를 구조화하여 효율적으로 관리합니다.

📚 학술 정의

관계형 데이터 모델은 E.F. Codd가 1970년 "A Relational Model of Data for Large Shared Data Banks"에서 제안했습니다.

핵심 개념:

  • 릴레이션(Relation): 수학의 집합론에 기반한 2차원 표. R ⊆ D₁ × D₂ × ... × Dₙ
  • 튜플(Tuple): 표의 한 행(row). 속성값들의 순서쌍 집합
  • 속성(Attribute): 표의 한 열(column). 특정 도메인의 값을 가짐
  • 도메인(Domain): 속성이 취할 수 있는 값들의 집합 (예: 학번 도메인 = {001, 002, ...})
  • 카디날리티(Cardinality): 릴레이션의 행(튜플) 개수
  • 디그리(Degree): 릴레이션의 열(속성) 개수

관계형 모델의 강점은 수학적 엄밀성과 쿼리 최적화에 있습니다.

1.2 핵심 개념: 테이블, 행, 열

🎯 쉽게 이해하기

테이블은 행(row)과 열(column)로 이루어진 2차원 구조입니다.

  • 열(Column): 특정 속성을 나타냅니다. 예) "이름" 열, "학번" 열
  • 행(Row): 하나의 객체에 대한 정보입니다. 예) 학생 한 명의 모든 정보
  • 셀: 행과 열이 만나는 지점의 하나의 값

학생 테이블 예시:

학번 이름 학년 전공
001 김철수 2 컴퓨터공학
002 이영희 3 경영학
003 박민준 1 수학

📚 형식적 정의

관계 스키마(Relation Schema): R(A₁:D₁, A₂:D₂, ..., Aₙ:Dₙ) 여기서 각 Aᵢ는 속성명, Dᵢ는 도메인입니다.

관계 인스턴스(Relation Instance): 스키마에 대한 실제 데이터 튜플들의 집합.

관계대수(Relational Algebra) 기본 연산자:

  • 선택(Selection, σ): σ_{condition}(R) - 조건을 만족하는 행 선택. 예) σ_{grade>2}(Students)
  • 프로젝션(Projection, π): π_{A₁,A₂,...}(R) - 특정 속성만 선택. 예) π_{name,dept}(Students)
  • 자연 조인(Natural Join, ⋈): R₁ ⋈ R₂ - 공통 속성을 기준으로 두 릴레이션 결합
  • 카티션 곱(Cartesian Product, ×): R₁ × R₂ - 모든 조합의 곱집합
  • 합집합(Union, ∪): R₁ ∪ R₂ - 두 릴레이션의 모든 행
  • 차집합(Difference, −): R₁ − R₂ - R₁에는 있고 R₂에 없는 행
  • 교집합(Intersection, ∩): R₁ ∩ R₂ = R₁ − (R₁ − R₂)

1.3 키(Key)의 종류

🎯 쉽게 이해하기

학생들을 구분하기 위해 학번이 필요하죠. 이것이 기본 키(Primary Key)입니다.

키의 종류를 친구관계로 비유하면:

  • 기본 키(PK): 각 사람의 주민등록번호처럼 유일하게 구분해주는 것
  • 외래 키(FK): 친구 목록에서 친구가 누구인지 가리키는 포인터. 다른 테이블과 연결하는 것

예시:

Students 테이블 - 학번이 PK (각 학생을 유일하게 식별)

Enrollments 테이블 - 학번과 과목코드가 FK (학생과 강좌를 연결)

📚 형식적 정의

K를 릴레이션 R의 속성들의 부분집합이라 하자.

  • 슈퍼키(Superkey): K의 어떤 두 튜플도 모든 속성에서 동일한 값을 가지지 않음. ∀t₁,t₂ ∈ r, t₁ ≠ t₂ ⟹ t₁[K] ≠ t₂[K]
  • 후보 키(Candidate Key): 최소 슈퍼키. 어떤 진부분집합도 슈퍼키가 아님
  • 기본 키(Primary Key): DBA가 선택한 후보 키. NULL이 될 수 없음(NOT NULL, UNIQUE)
  • 대체 키(Alternate Key): 선택되지 않은 후보 키들
  • 외래 키(Foreign Key): 릴레이션 R₁의 속성 FK는 다른 릴레이션 R₂의 기본 키를 참조. ∀t ∈ R₁, t[FK] ∈ {t'[PK] : t' ∈ R₂} ∪ {NULL}

무결성 제약조건(Integrity Constraints):

  • 엔티티 무결성(Entity Integrity): 기본 키는 NULL일 수 없음
  • 참조 무결성(Referential Integrity): 외래 키는 참조 대상의 기본 키 값이거나 NULL
  • 도메인 무결성(Domain Integrity): 각 속성은 도메인 내의 값만 가짐

1.4 관계 연산

🎯 쉽게 이해하기

데이터베이스에서 데이터를 꺼낼 때 사용하는 기본 동작들입니다.

  • 필터링(σ - Selection): "학년이 2학년인 학생만 보여줘" → 특정 행만 추출
  • 특정 열만 보기(π - Projection): "학번과 이름만 보여줘" → 특정 열만 추출
  • 두 표 합치기(⋈ - Join): 학생 정보와 수강 정보를 학번으로 묶어서 보여주기
  • 합치기(∪ - Union): 컴퓨터공학 학생들과 경영학 학생들 합치기

실제 예시:

학생 테이블에서 "2학년 학생의 이름과 전공만 보고 싶다" →

π_{name,dept}(σ_{grade=2}(Students))

📚 관계대수 상세

기본 집합 연산:

  • 합집합(∪): r₁ ∪ r₂ = {t | t ∈ r₁ ∨ t ∈ r₂}. 호환성: 같은 스키마 필요
  • 교집합(∩): r₁ ∩ r₂ = r₁ − (r₁ − r₂)
  • 차집합(−): r₁ − r₂ = {t | t ∈ r₁ ∧ t ∉ r₂}
  • 카티션 곱(×): r₁ × r₂ = {(t₁, t₂) | t₁ ∈ r₁ ∧ t₂ ∈ r₂}. 결과 도수 = |r₁| × |r₂|

관계식 연산:

  • 선택(σ): σ_F(r) = {t | t ∈ r ∧ F(t) = true}. 여기서 F는 선택 조건식
  • 프로젝션(π): π_{A₁,...,Aₖ}(r) = {[t[A₁],...,t[Aₖ]] | t ∈ r}. 중복 제거됨
  • 조인(⋈): r₁ ⋈_F r₂ = σ_F(r₁ × r₂). 특수한 경우가 내부 조인, 외부 조인
  • 나누기(÷): r₁ ÷ r₂ = {t | ∀t' ∈ r₂, (t ⊕ t') ∈ r₁}. 복잡한 쿼리 표현에 사용
  • 이름변경(ρ): ρ_{S(B₁,...,Bₖ)}(r) - 릴레이션 이름이나 속성명 변경

동치 변환 규칙(Equivalence Rules):

  • σ_{F₁∧F₂}(r) ≡ σ_{F₁}(σ_{F₂}(r)) (선택의 분해)
  • π_{A}(σ_F(r)) ≡ σ_F(π_{A}(r)) (선택과 프로젝션의 순서교환은 조건부)
  • σ_{F₁}(r₁ ⋈ r₂) ≡ σ_{F₁}(r₁) ⋈ r₂ (선택 연산 푸시다운)
