6장. NoSQL

1. NoSQL이란?

쉬운 설명

관계형 DB는 모든 데이터를 테이블(행과 열)에 넣으려고 합니다. 하지만 모든 데이터가 표 형식에 맞지는 않습니다:

  • SNS 게시글: 작성자, 내용, 사진 여러 장, 댓글(개수 미정)...
  • 쇼핑몰 상품: 가구는 "가로×세로×높이", 의류는 "사이즈 옵션", 음식은 "유통기한"...
  • 사용자 프로필: 누군가는 25개의 취미, 누군가는 1개...

NoSQL: "Not Only SQL". 표 형식 대신 더 자유로운 형태로 저장하는 데이터베이스입니다.

논문 수준 설명

NoSQL의 등장: 2000년대 웹 서비스 확산으로 빅데이터 처리 필요성 증가. 기존 RDBMS의 확장성(Scalability) 한계 극복을 위해 등장.

특징:

  • 스키마리스(Schema-less): 사전에 구조 정의 불필요
  • 수평 확장(Horizontal Scaling): 여러 서버에 분산 저장 용이
  • 높은 처리량(High Throughput)
  • 느슨한 일관성(Eventual Consistency)

CAP 정리 예고: Consistency, Availability, Partition tolerance 중 2개만 선택 가능

2. Document DB (MongoDB)

쉬운 설명

서류 봉투를 생각해봅시다. 봉투 안에는 여권, 사진, 메모, 뭐든 들어갈 수 있습니다.

Document DB는 JSON 형태로 다양한 데이터를 한곳에 담을 수 있습니다:

{
  "이름": "김철수",
  "나이": 30,
  "취미": ["축구", "영화"],
  "주소": {
    "시": "서울",
    "구": "강남구"
  }
}

모든 필드가 필수일 필요 없음. 어떤 사람은 "직업" 필드가 있고, 어떤 사람은 없을 수 있습니다.

논문 수준 설명

BSON (Binary JSON): MongoDB의 저장 형식. JSON과 호환되지만 바이너리 인코딩으로 더 효율적.

스키마리스의 장단점:

  • 장점: 유연성, 빠른 개발 속도
  • 단점: 데이터 무결성 보장 어려움, 쿼리 복잡성 증가

내장(Embedding) vs 참조(Referencing):

  • 내장: 관련 데이터를 한 문서 안에 포함 (중복 가능, 한 번에 조회)
  • 참조: 다른 문서의 ID를 참조 (정규화, 여러 번 조회 필요)

트랜잭션: MongoDB 4.0+에서 멀티 문서 트랜잭션 지원 (다만 RDBMS보다 제한적)

3. Key-Value & Column-Family

쉬운 설명

Key-Value DB: 사물함처럼 번호(키)로 물건(값)을 꺼냅니다. "123번 사물함에 뭐가 있지?" → 바로 꺼냄.

  • 예: Redis, Memcached
  • 용도: 캐시, 세션 저장, 실시간 순위판

Column-Family DB: 거대한 스프레드시트. 수억 개의 행을 빠르게 처리합니다.

  • 예: Cassandra, HBase
  • 용도: 시계열 데이터(시간에 따른 센서 값), 로그 저장

논문 수준 설명

Redis 데이터 구조: String, List, Set, Sorted Set, Hash, Stream 등을 지원하는 In-Memory 저장소.

Cassandra의 파티션 전략: 일관된 해싱(Consistent Hashing)을 사용하여 데이터를 여러 노드에 분산. 한 노드 장애 시 다른 노드로 빠르게 재조정.

LSM-Tree (Log-Structured Merge-Tree):

  • 쓰기 연산 최적화
  • 메모리의 정렬된 맵(MemTable) → 디스크의 정렬된 문자열 테이블(SSTable)로 쓰기
  • 읽기 시 여러 SSTable을 병합 검색

Cassandra의 일관성 모델: Eventual Consistency. 쓰기 후 일부 레플리카는 즉시 최신값 보장 불가.

4. Graph DB

쉬운 설명

인맥 지도를 그려봅시다:

  • 노드: 사람 (김철수, 이영희...)
  • 간선: 관계 (친구, 직장 동료, 친척...)

이 구조에서 "김철수와 가장 가까운 사람은?" 또는 "친구의 친구 중 같은 취미를 가진 사람은?"을 빠르게 찾을 수 있습니다.

예: Facebook의 친구 추천, Netflix의 콘텐츠 추천, Google Maps의 경로 검색

논문 수준 설명

속성 그래프 모델(Property Graph Model):

  • 노드: 엔티티. name, age 등의 속성 가짐
  • 간선: 관계. relation_type, weight 등의 속성 가짐

Cypher 쿼리 언어: Neo4j의 표준 쿼리 언어. SQL과 유사하지만 그래프 패턴 매칭에 최적화.

MATCH (a:Person)-[:FRIEND]-(b:Person)
WHERE a.name = '김철수'
RETURN b.name

그래프 순회 알고리즘: BFS (Breadth-First Search), DFS (Depth-First Search), Dijkstra (최단 경로), PageRank 등
