2장. SQL 핵심

2.1 SQL이란?

🎯 쉽게 이해하기

SQL(Structured Query Language)은 데이터베이스와 대화하는 언어입니다. 마치 우리가 한국어로 대화하듯이, SQL로 데이터베이스에게 명령을 내릴 수 있어요.

예시:

SELECT name FROM students WHERE grade = 2;

위 명령은 "students 테이블에서 grade가 2인 학생의 name을 보여줘"라는 뜻입니다.

📚 학술 정의

SQL은 1974년 Donald Chamberlin과 Raymond Boyce가 IBM에서 SEQUEL(Structured English Query Language)로 처음 개발했으며, 이후 SQL(Structured Query Language)로 표준화되었습니다. 1986년에 ANSI(American National Standards Institute)에서 표준을 공식화했고, 1987년에 ISO(International Organization for Standardization)에 채택되었습니다.

SQL의 특징:

  • 선언적(Declarative): "무엇을" 할지 명시하지만, "어떻게" 할지는 DBMS가 결정
  • 절차적이 아님(Non-procedural): 단계별 알고리즘을 지정하지 않음
  • 집합 지향(Set-oriented): 한 번에 여러 행을 처리

SQL의 세 가지 부분(Three Sub-languages):

  • DDL(Data Definition Language): CREATE, ALTER, DROP. 구조 정의
  • DML(Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE. 데이터 조작
  • DCL(Data Control Language): GRANT, REVOKE. 권한 관리

2.2 데이터 정의(DDL)

🎯 쉽게 이해하기

DDL은 데이터베이스의 구조를 만들고 수정하는 명령어들입니다.

  • CREATE TABLE: 새로운 테이블 만들기
  • ALTER TABLE: 기존 테이블 수정하기 (열 추가, 삭제, 타입 변경 등)
  • DROP TABLE: 테이블 삭제하기

CREATE TABLE 예시:

CREATE TABLE students (
  student_id INT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  grade INT,
  department VARCHAR(30)
);

📚 형식적 정의

CREATE TABLE 구문: 

CREATE TABLE TableName (
  ColName1 DataType Constraint,
  ColName2 DataType Constraint,
  ...
  TableConstraint
);

데이터 타입(주요):

  • INT, BIGINT: 정수
  • FLOAT, DECIMAL(p,s): 실수 (p=전체자리수, s=소수자리수)
  • VARCHAR(n), CHAR(n): 문자열 (n=최대 길이)
  • DATE, TIMESTAMP: 날짜/시간
  • BOOLEAN: TRUE/FALSE

제약조건(Constraints):

  • NOT NULL: 반드시 값이 있어야 함
  • UNIQUE: 같은 값이 중복되지 않음
  • PRIMARY KEY: NOT NULL + UNIQUE + 기본 식별자
  • FOREIGN KEY: 다른 테이블의 PRIMARY KEY 참조
  • CHECK: 조건식이 참인 값만 허용
  • DEFAULT: 지정되지 않으면 기본값 사용

ALTER TABLE 예시:

ALTER TABLE students ADD COLUMN email VARCHAR(100);
ALTER TABLE students MODIFY COLUMN grade INT NOT NULL;
ALTER TABLE students DROP COLUMN email;

2.3 데이터 조작(DML)

🎯 쉽게 이해하기

DML은 테이블의 데이터를 추가, 조회, 수정, 삭제하는 명령어들입니다. CRUD라고도 부릅니다:

  • Create = INSERT: 새로운 데이터 추가
  • Read = SELECT: 데이터 조회
  • Update = UPDATE: 기존 데이터 수정
  • Delete = DELETE: 데이터 삭제

SELECT 기본 구조:

SELECT 열1, 열2
FROM 테이블
WHERE 조건
ORDER BY 열 ASC|DESC;

JOIN으로 두 테이블 연결하기:

학생의 정보와 그들이 수강하는 과목 정보를 함께 보고 싶을 때 JOIN을 사용합니다.

SELECT s.name, c.course_name
FROM students s
INNER JOIN enrollments e ON s.student_id = e.student_id
INNER JOIN courses c ON e.course_id = c.course_id;

📚 형식적 정의

SELECT 쿼리 처리 순서(Execution Order):

  1. FROM: 데이터 소스 결정
  2. WHERE: 행 필터링 (T ⊆ R)
  3. GROUP BY: 그룹 분할
  4. HAVING: 그룹 필터링
  5. SELECT: 열 선택 및 계산
  6. DISTINCT: 중복 제거
  7. ORDER BY: 정렬
  8. LIMIT/OFFSET: 페이징

JOIN의 유형:

  • INNER JOIN: R₁ ⋈ R₂ = {(t₁,t₂) | t₁ ∈ R₁ ∧ t₂ ∈ R₂ ∧ 조건}. 공통 부분만
  • LEFT OUTER JOIN: R₁의 모든 행 + 매치되는 R₂ 행들
  • RIGHT OUTER JOIN: R₂의 모든 행 + 매치되는 R₁ 행들
  • FULL OUTER JOIN: R₁과 R₂의 모든 행
  • CROSS JOIN: R₁ × R₂ (카티션 곱)

집계 함수(Aggregate Functions):

  • COUNT(expr): NULL이 아닌 행 개수. COUNT(*) = 전체 행 개수
  • SUM(expr): ∑ expr
  • AVG(expr): (∑ expr) / COUNT(expr)
  • MIN(expr), MAX(expr): 최소값, 최대값
  • GROUP_CONCAT(expr): 문자열 연결

윈도우 함수(Window Functions):

SELECT
  name,
  salary,
  ROW_NUMBER() OVER (ORDER BY salary DESC) AS rank,
  LAG(salary) OVER (ORDER BY salary DESC) AS prev_salary,
  LEAD(salary) OVER (ORDER BY salary DESC) AS next_salary
FROM employees;

2.4 서브쿼리와 집계

🎯 쉽게 이해하기

서브쿼리 = "질문 안에 질문" 넣기

예를 들어, "평균 학점 이상인 학생을 찾아줘"라는 요청은:

SELECT name
FROM students
WHERE grade >= (SELECT AVG(grade) FROM students);

집계 함수 = 많은 데이터를 요약하는 것

  • COUNT: 몇 명인가? → 50명
  • SUM: 총합은? → 총점 4500점
  • AVG: 평균은? → 평균 학점 3.5
  • MAX/MIN: 최고/최저는? → 최고 학점 4.5

GROUP BY로 그룹 나누기:

"전공별로 학생 수를 세어줘"

SELECT department, COUNT(*) as student_count
FROM students
GROUP BY department;

📚 형식적 정의

서브쿼리(Subquery, Inner Query): 다른 쿼리 내에 포함된 SELECT 문.

상관 서브쿼리(Correlated Subquery):

SELECT e1.name, e1.salary
FROM employees e1
WHERE e1.salary > (
  SELECT AVG(e2.salary)
  FROM employees e2
  WHERE e2.department = e1.department
);

외부 쿼리의 각 행에 대해 서브쿼리가 실행됨. 성능이 낮을 수 있음.

IN/EXISTS 연산자:

  • IN: WHERE col IN (SELECT ...)
  • EXISTS: WHERE EXISTS (SELECT ... FROM ... WHERE ...)
  • NOT IN, NOT EXISTS: 반대 조건

윈도우 함수의 구조:

function_name(expr) OVER (
  [PARTITION BY col1, col2, ...]
  [ORDER BY col3 [ASC|DESC]]
  [ROWS|RANGE BETWEEN ...]
)
  • ROW_NUMBER(): 순위 (중복 허용 안 함)
  • RANK(): 순위 (중복 허용)
  • DENSE_RANK(): 순위 (밀집)
  • LAG(col, offset): 이전 행의 값
  • LEAD(col, offset): 다음 행의 값
  • FIRST_VALUE(), LAST_VALUE(): 첫/마지막 값

하 · 입문

한 줄로 이해하기

SQL은 데이터베이스에 "이런 데이터를 보여줘", "이렇게 바꿔줘"라고 부탁하는 영어 비슷한 언어입니다.

왜 중요한가?

SQL은 50년 넘게 살아남은 "데이터 세계의 영어"입니다. 첫째, 매출 데이터에서 어제 가장 많이 팔린 상품을 단 한 줄의 SELECT로 뽑을 수 있습니다. 둘째, 회원가입 폼이 INSERT 문 한 줄로 데이터베이스에 저장됩니다. 셋째, 잘못 입력된 전화번호를 UPDATE 한 번으로 수만 건 일괄 수정할 수 있습니다.

🌱 핵심 용어

  • SELECT: 데이터를 꺼내 보는 명령. "보여줘"의 뜻.
  • INSERT: 새로운 행을 추가하는 명령.
  • UPDATE: 기존 행의 값을 바꾸는 명령.
  • DELETE: 행을 지우는 명령. WHERE 없이 쓰면 다 지워집니다.
  • WHERE: 조건절. "이런 행만"을 지정합니다.

개념 풀어쓰기

SQL은 마치 식당에서 주문서를 쓰는 것과 같습니다. 종업원에게 "오늘의 메뉴 중 매콤한 것만 보여 주세요. 가격은 1만 원 이하로요. 그리고 매운 정도 순으로 정렬해 주세요"라고 말하는 것과 SQL의 SELECT 문은 거의 같은 구조입니다. 무엇을 가져올지(SELECT 절), 어디서 가져올지(FROM 절), 어떤 조건인지(WHERE 절), 어떻게 정렬할지(ORDER BY 절)를 차례로 적습니다. 처음에는 영어 단어가 낯설어 보이지만, 사실 자연스러운 부탁의 순서입니다. INSERT는 새 손님을 명단에 적는 것, UPDATE는 적은 값을 고치는 것, DELETE는 명단에서 지우는 것입니다. 가장 중요한 점은 "한 번에 여러 행을 처리한다"는 것입니다. 한 줄을 일일이 고치는 게 아니라 "20세 미만 회원 모두"라는 식으로 묶음 단위로 명령합니다. 그래서 빠르고 안전합니다. 처음엔 SELECT 한 가지만 자유롭게 다룰 수 있어도 회사에서 요구하는 데이터 분석의 70%는 해낼 수 있습니다.

💡 학습 TIP: SQL은 책으로 배우기보다 직접 쳐 보는 게 가장 빠릅니다. SQLite나 온라인 SQL 연습 사이트에 작은 테이블 하나 만들고 SELECT를 50번만 쳐 보세요.

요약

  • SQL은 DB에 명령을 내리는 언어입니다.
  • SELECT는 조회, INSERT/UPDATE/DELETE는 변경.
  • WHERE 절로 대상을 좁힙니다.
  • ORDER BY로 정렬, LIMIT로 개수 제한.
  • 한 번에 여러 행을 묶음 처리하는 것이 강점입니다.
중 · 응용

이론 깊이

SQL은 선언적 언어입니다. "어떻게" 가져올지가 아니라 "무엇을" 가져올지를 기술하면, 옵티마이저가 가장 효율적인 실행계획을 만들어 냅니다. 따라서 같은 결과를 만드는 쿼리도 작성 방식에 따라 성능이 천차만별입니다. SQL 표준은 ANSI SQL로 정의되며 ISO에서도 관리하지만, 각 RDBMS는 자체 방언을 갖습니다. SQL의 큰 줄기는 DDL(Data Definition Language: CREATE, ALTER, DROP), DML(Data Manipulation Language: SELECT, INSERT, UPDATE, DELETE), DCL(Data Control Language: GRANT, REVOKE), TCL(Transaction Control: COMMIT, ROLLBACK)로 나뉩니다. 조인은 INNER, LEFT, RIGHT, FULL 네 가지가 표준이며, 자기 자신을 조인하는 셀프 조인은 조직도나 카테고리 트리 같은 계층 구조 표현에 쓰입니다. 그룹화는 GROUP BY와 HAVING이 핵심이고, 윈도우 함수(OVER 절)는 그룹화하지 않고도 누적·순위·이동평균을 계산하게 해 줍니다.

실전 SQL/쿼리

-- 카테고리별 매출 상위 3위까지 추출 (윈도우 함수)
WITH ranked AS (
  SELECT category,
         product_name,
         SUM(amount) AS total_sales,
         RANK() OVER (
           PARTITION BY category
           ORDER BY SUM(amount) DESC
         ) AS rnk
  FROM   sales
  WHERE  sold_at >= '2025-01-01'
  GROUP  BY category, product_name
)
SELECT category, product_name, total_sales
FROM   ranked
WHERE  rnk <= 3
ORDER  BY category, rnk;

-- 외부 조인으로 주문이 없는 회원도 포함
SELECT m.member_id, m.name, COUNT(o.order_id) AS order_cnt
FROM   member m
LEFT   JOIN orders o ON m.member_id = o.member_id
GROUP  BY m.member_id, m.name;

예제

💻 예제 1: 월별 활성 사용자(MAU) 계산

로그 테이블에서 DATE_TRUNC('month', logged_at)로 월 단위 그룹을 만들고 COUNT(DISTINCT user_id)로 중복을 제거합니다. 마케팅 대시보드의 가장 기본 지표입니다.

💻 예제 2: 코호트 잔존율

가입 월별 회원을 묶고, 이후 N개월차에 활동한 비율을 계산합니다. CASE WHEN과 GROUP BY 조합으로 코호트 매트릭스를 만들 수 있습니다.

실무 응용

광고 분석 시스템에서는 일별 로그가 수억 건 쌓이며, GROUP BY와 윈도우 함수가 ETL 파이프라인의 중심이 됩니다. 또한 결제 정산 시스템은 트랜잭션 안에서 INSERT와 UPDATE를 함께 실행해 일관성을 보장합니다. BI 도구(Looker, Metabase)는 SQL을 자동 생성해 비개발자도 분석할 수 있게 해 주며, 데이터 엔지니어는 dbt로 SQL 모델을 버전 관리합니다.

⚠️ 자주 하는 실수: WHERE 절에 함수를 씌우면 인덱스를 못 타는 경우가 많습니다. WHERE DATE(created_at) = '2025-01-01' 대신 범위 비교(BETWEEN)를 쓰세요. 또한 SELECT *는 운영 환경에서 피해야 할 1순위 습관입니다.
상 · 심화

전문가 레벨

SQL 옵티마이저는 파서, 리라이터, 플래너, 익스큐터 단계로 쿼리를 처리합니다. 플래너는 통계(pg_statistic, dba_tab_columns)를 바탕으로 비용 모델을 계산해 Nested Loop, Hash Join, Merge Join 중 최적을 선택합니다. 옵티마이저 힌트(Oracle의 /*+ INDEX(...) */, MySQL의 USE INDEX)는 마지막 보루이며, 본질적으로는 통계 갱신과 인덱스 설계가 우선입니다. 실행계획에서 자주 마주치는 함정으로는 함수 인덱스 미사용, 암묵적 형변환으로 인한 풀스캔, 카디널리티 추정 오류가 있습니다. CTE(Common Table Expression)는 PostgreSQL 12 이전에는 옵티마이저 펜스로 작용해 별도 임시결과를 만들었지만, 12 이후로는 인라이닝이 기본입니다. 윈도우 함수의 프레임 절(ROWS, RANGE, GROUPS)은 정확한 의미가 다르며, 시계열 분석에서는 GROUPS 모드가 동률을 묶어 처리합니다. 재귀 CTE는 그래프 탐색과 BOM(자재 명세서) 전개에 강력하며, UNION ALL 형태로 종료 조건을 반드시 명시해야 무한 루프를 피할 수 있습니다.

고급 패턴

-- 재귀 CTE로 조직도 계층 탐색
WITH RECURSIVE org_tree AS (
  SELECT employee_id, manager_id, name, 1 AS depth
  FROM   employee
  WHERE  manager_id IS NULL  -- CEO
  UNION  ALL
  SELECT e.employee_id, e.manager_id, e.name, t.depth + 1
  FROM   employee e
  JOIN   org_tree t ON e.manager_id = t.employee_id
)
SELECT depth, name FROM org_tree ORDER BY depth, name;

-- LATERAL 조인으로 회원별 최근 주문 N건
SELECT m.member_id, m.name, recent.order_id, recent.amount
FROM   member m
CROSS  JOIN LATERAL (
  SELECT order_id, amount
  FROM   orders o
  WHERE  o.member_id = m.member_id
  ORDER  BY o.created_at DESC
  LIMIT  3
) recent;

최신 동향

BigQuery, Snowflake, Databricks는 SQL을 분산 분석 엔진의 표준 인터페이스로 채택했습니다. 또한 DuckDB는 단일 노드에서 컬럼나 분석을 수행하는 임베디드 OLAP 엔진으로 데이터 사이언스 영역에서 빠르게 확산 중입니다. SQL/JSON 표준은 JSON 데이터를 1급 시민으로 다루며, PostgreSQL의 JSONB와 jsonpath가 대표 구현입니다. LLM 기반 Text-to-SQL(예: Vanna, sqlcoder)도 떠오르는 분야로, 자연어 질의를 SQL로 변환합니다.

관련 분야

  • 쿼리 옵티마이저 이론(Volcano/Cascades 프레임워크).
  • 컬럼나 스토리지와 벡터화 실행.
  • 스트리밍 SQL(Flink SQL, ksqlDB).
