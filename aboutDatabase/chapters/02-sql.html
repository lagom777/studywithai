2장. SQL 핵심

2.1 SQL이란?

🎯 쉽게 이해하기

SQL(Structured Query Language)은 데이터베이스와 대화하는 언어입니다. 마치 우리가 한국어로 대화하듯이, SQL로 데이터베이스에게 명령을 내릴 수 있어요.

예시:

SELECT name FROM students WHERE grade = 2;

위 명령은 "students 테이블에서 grade가 2인 학생의 name을 보여줘"라는 뜻입니다.

📚 학술 정의

SQL은 1974년 Donald Chamberlin과 Raymond Boyce가 IBM에서 SEQUEL(Structured English Query Language)로 처음 개발했으며, 이후 SQL(Structured Query Language)로 표준화되었습니다. 1986년에 ANSI(American National Standards Institute)에서 표준을 공식화했고, 1987년에 ISO(International Organization for Standardization)에 채택되었습니다.

SQL의 특징:

  • 선언적(Declarative): "무엇을" 할지 명시하지만, "어떻게" 할지는 DBMS가 결정
  • 절차적이 아님(Non-procedural): 단계별 알고리즘을 지정하지 않음
  • 집합 지향(Set-oriented): 한 번에 여러 행을 처리

SQL의 세 가지 부분(Three Sub-languages):

  • DDL(Data Definition Language): CREATE, ALTER, DROP. 구조 정의
  • DML(Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE. 데이터 조작
  • DCL(Data Control Language): GRANT, REVOKE. 권한 관리

2.2 데이터 정의(DDL)

🎯 쉽게 이해하기

DDL은 데이터베이스의 구조를 만들고 수정하는 명령어들입니다.

  • CREATE TABLE: 새로운 테이블 만들기
  • ALTER TABLE: 기존 테이블 수정하기 (열 추가, 삭제, 타입 변경 등)
  • DROP TABLE: 테이블 삭제하기

CREATE TABLE 예시:

CREATE TABLE students (
  student_id INT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  grade INT,
  department VARCHAR(30)
);

📚 형식적 정의

CREATE TABLE 구문: 

CREATE TABLE TableName (
  ColName1 DataType Constraint,
  ColName2 DataType Constraint,
  ...
  TableConstraint
);

데이터 타입(주요):

  • INT, BIGINT: 정수
  • FLOAT, DECIMAL(p,s): 실수 (p=전체자리수, s=소수자리수)
  • VARCHAR(n), CHAR(n): 문자열 (n=최대 길이)
  • DATE, TIMESTAMP: 날짜/시간
  • BOOLEAN: TRUE/FALSE

제약조건(Constraints):

  • NOT NULL: 반드시 값이 있어야 함
  • UNIQUE: 같은 값이 중복되지 않음
  • PRIMARY KEY: NOT NULL + UNIQUE + 기본 식별자
  • FOREIGN KEY: 다른 테이블의 PRIMARY KEY 참조
  • CHECK: 조건식이 참인 값만 허용
  • DEFAULT: 지정되지 않으면 기본값 사용

ALTER TABLE 예시:

ALTER TABLE students ADD COLUMN email VARCHAR(100);
ALTER TABLE students MODIFY COLUMN grade INT NOT NULL;
ALTER TABLE students DROP COLUMN email;

2.3 데이터 조작(DML)

🎯 쉽게 이해하기

DML은 테이블의 데이터를 추가, 조회, 수정, 삭제하는 명령어들입니다. CRUD라고도 부릅니다:

  • Create = INSERT: 새로운 데이터 추가
  • Read = SELECT: 데이터 조회
  • Update = UPDATE: 기존 데이터 수정
  • Delete = DELETE: 데이터 삭제

SELECT 기본 구조:

SELECT 열1, 열2
FROM 테이블
WHERE 조건
ORDER BY 열 ASC|DESC;

JOIN으로 두 테이블 연결하기:

학생의 정보와 그들이 수강하는 과목 정보를 함께 보고 싶을 때 JOIN을 사용합니다.

SELECT s.name, c.course_name
FROM students s
INNER JOIN enrollments e ON s.student_id = e.student_id
INNER JOIN courses c ON e.course_id = c.course_id;

📚 형식적 정의

SELECT 쿼리 처리 순서(Execution Order):

  1. FROM: 데이터 소스 결정
  2. WHERE: 행 필터링 (T ⊆ R)
  3. GROUP BY: 그룹 분할
  4. HAVING: 그룹 필터링
  5. SELECT: 열 선택 및 계산
  6. DISTINCT: 중복 제거
  7. ORDER BY: 정렬
  8. LIMIT/OFFSET: 페이징

JOIN의 유형:

  • INNER JOIN: R₁ ⋈ R₂ = {(t₁,t₂) | t₁ ∈ R₁ ∧ t₂ ∈ R₂ ∧ 조건}. 공통 부분만
  • LEFT OUTER JOIN: R₁의 모든 행 + 매치되는 R₂ 행들
  • RIGHT OUTER JOIN: R₂의 모든 행 + 매치되는 R₁ 행들
  • FULL OUTER JOIN: R₁과 R₂의 모든 행
  • CROSS JOIN: R₁ × R₂ (카티션 곱)

집계 함수(Aggregate Functions):

  • COUNT(expr): NULL이 아닌 행 개수. COUNT(*) = 전체 행 개수
  • SUM(expr): ∑ expr
  • AVG(expr): (∑ expr) / COUNT(expr)
  • MIN(expr), MAX(expr): 최소값, 최대값
  • GROUP_CONCAT(expr): 문자열 연결

윈도우 함수(Window Functions):

SELECT
  name,
  salary,
  ROW_NUMBER() OVER (ORDER BY salary DESC) AS rank,
  LAG(salary) OVER (ORDER BY salary DESC) AS prev_salary,
  LEAD(salary) OVER (ORDER BY salary DESC) AS next_salary
FROM employees;

2.4 서브쿼리와 집계

🎯 쉽게 이해하기

서브쿼리 = "질문 안에 질문" 넣기

예를 들어, "평균 학점 이상인 학생을 찾아줘"라는 요청은:

SELECT name
FROM students
WHERE grade >= (SELECT AVG(grade) FROM students);

집계 함수 = 많은 데이터를 요약하는 것

  • COUNT: 몇 명인가? → 50명
  • SUM: 총합은? → 총점 4500점
  • AVG: 평균은? → 평균 학점 3.5
  • MAX/MIN: 최고/최저는? → 최고 학점 4.5

GROUP BY로 그룹 나누기:

"전공별로 학생 수를 세어줘"

SELECT department, COUNT(*) as student_count
FROM students
GROUP BY department;

📚 형식적 정의

서브쿼리(Subquery, Inner Query): 다른 쿼리 내에 포함된 SELECT 문.

상관 서브쿼리(Correlated Subquery):

SELECT e1.name, e1.salary
FROM employees e1
WHERE e1.salary > (
  SELECT AVG(e2.salary)
  FROM employees e2
  WHERE e2.department = e1.department
);

외부 쿼리의 각 행에 대해 서브쿼리가 실행됨. 성능이 낮을 수 있음.

IN/EXISTS 연산자:

  • IN: WHERE col IN (SELECT ...)
  • EXISTS: WHERE EXISTS (SELECT ... FROM ... WHERE ...)
  • NOT IN, NOT EXISTS: 반대 조건

윈도우 함수의 구조:

function_name(expr) OVER (
  [PARTITION BY col1, col2, ...]
  [ORDER BY col3 [ASC|DESC]]
  [ROWS|RANGE BETWEEN ...]
)
  • ROW_NUMBER(): 순위 (중복 허용 안 함)
  • RANK(): 순위 (중복 허용)
  • DENSE_RANK(): 순위 (밀집)
  • LAG(col, offset): 이전 행의 값
  • LEAD(col, offset): 다음 행의 값
  • FIRST_VALUE(), LAST_VALUE(): 첫/마지막 값
