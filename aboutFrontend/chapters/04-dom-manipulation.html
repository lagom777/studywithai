4장. DOM 조작

Document Object Model의 개념과 JavaScript를 통한 동적 조작

4.1 DOM이란?

DOM(Document Object Model)은 HTML 문서를 객체 트리 구조로 표현한 API입니다. 브라우저가 HTML을 파싱할 때 각 요소를 트리 형태로 구성하여 JavaScript에서 접근하고 조작할 수 있게 해줍니다.

DOM 트리 구조 시각화

아래의 3D 시각화에서 DOM 트리를 입체적으로 확인할 수 있습니다. 마우스로 각 노드에 호버하면 해당 요소가 강조됩니다.

💡 팁: 마우스를 움직여 각 노드를 하이라이트하세요. 파란색=div, 초록색=p, 주황색=span

const root = document.documentElement; // <html> 루트
const body = document.body; // <body> 요소
const nodes = document.childNodes; // 자식 노드 목록

4.2 요소 선택과 탐색

DOM 조작의 첫 단계는 조작할 요소를 선택하는 것입니다. 다양한 선택 메서드와 탐색 방법이 있습니다.

선택 메서드

// 선택 메서드들
document.getElementById('id') // ID로 선택 (가장 빠름)
document.querySelector('selector') // CSS 선택자로 첫 요소 선택
document.querySelectorAll('selector') // CSS 선택자로 모든 요소 선택
document.getElementsByClassName('cls') // 클래스로 선택
document.getElementsByTagName('div') // 태그명으로 선택

노드 탐색

const elem = document.querySelector('.target');
elem.parentNode; // 부모 노드
elem.childNodes; // 자식 노드 목록 (텍스트 노드 포함)
elem.children; // 자식 요소 (요소만)
elem.firstChild; // 첫 자식
elem.nextSibling; // 다음 형제 노드
elem.previousSibling; // 이전 형제 노드

라이브 DOM 트리 뷰어

버튼을 클릭하여 DOM 구조를 동적으로 변경하고 실시간으로 트리를 확인하세요.

4.3 이벤트 시스템

이벤트는 사용자 상호작용(클릭, 입력 등)이나 브라우저 작동에 의해 발생하는 신호입니다. JavaScript는 이벤트를 감지하고 처리합니다.

이벤트 리스너 등록

element.addEventListener('click', (event) => {
  console.log('클릭됨!', event);
});

element.removeEventListener('click', handler);

이벤트 버블링과 캡처링

버블링은 이벤트가 발생한 요소에서 시작하여 부모 요소로 전파되는 것입니다. 캡처링은 루트에서 시작하여 하위 요소로 전파됩니다.

이벤트 버블링 데모

이벤트 위임 (Event Delegation)

많은 자식 요소에 각각 리스너를 등록하는 대신, 부모 요소에 하나의 리스너를 등록하고 event.target을 확인하는 방식입니다.

parent.addEventListener('click', (event) => {
  if (event.target.matches('.child-item')) {
    // 자식 요소 클릭 처리
  }
});

4.4 DOM 조작 메서드

요소를 선택한 후, 내용을 변경하거나 새로운 요소를 추가할 수 있습니다.

메서드 설명 성능
innerHTML 요소의 HTML 내용을 문자열로 설정/조회 느림 (리파싱 발생)
textContent 요소의 텍스트만 설정/조회 (태그 미파싱) 빠름
createElement() 새로운 요소 생성 빠름
appendChild() 자식 요소 추가 중간
insertBefore() 특정 위치에 요소 삽입 중간
removeChild() 자식 요소 제거 빠름
// 요소 생성 및 추가
const newDiv = document.createElement('div');
newDiv.textContent = '새로운 요소';
newDiv.className = 'new-item';
parent.appendChild(newDiv);

// 요소 제거
parent.removeChild(newDiv);

// HTML 직접 변경 (주의: XSS 위험)
element.innerHTML = '<p>새 내용</p>';

4.5 가상 DOM 개념

실제 DOM 조작은 비용이 높으므로 (리플로우, 리페인트), React 같은 라이브러리는 가상 DOM을 사용합니다.

가상 DOM의 장점

  • 배칭(Batching): 여러 업데이트를 모아서 한 번에 적용
  • Diffing: 변경된 부분만 찾아 효율적으로 업데이트
  • 최적화: 불필요한 리렌더링 방지
  • 추상화: 개발자가 선언적 코드 작성 가능

실제 DOM vs 가상 DOM

1000개의 리스트 항목을 업데이트할 때의 차이를 시뮬레이션합니다.

Diffing 알고리즘

// 간단한 Diffing 개념
function diff(oldVDOM, newVDOM) {
  const changes = [];
  if (oldVDOM.type !== newVDOM.type) {
    changes.push({ action: 'replace', element: newVDOM });
  }
  // props, children 비교 ...
  return changes;
}

핵심 요약

  • ✅ DOM은 HTML 문서의 객체 표현이며 트리 구조로 되어있습니다.
  • ✅ querySelector, getElementById 등으로 요소를 선택하고 탐색합니다.
  • ✅ addEventListener로 이벤트를 감지하고 event 객체를 활용합니다.
  • ✅ createElement, appendChild 등으로 동적으로 요소를 조작합니다.
  • ✅ 가상 DOM은 성능 최적화를 위해 실제 DOM 업데이트를 스케줄링합니다.
