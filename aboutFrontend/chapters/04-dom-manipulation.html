4장. DOM 조작

Document Object Model의 개념과 JavaScript를 통한 동적 조작

4.1 DOM이란?

DOM(Document Object Model)은 HTML 문서를 객체 트리 구조로 표현한 API입니다. 브라우저가 HTML을 파싱할 때 각 요소를 트리 형태로 구성하여 JavaScript에서 접근하고 조작할 수 있게 해줍니다.

DOM 트리 구조 시각화

아래의 3D 시각화에서 DOM 트리를 입체적으로 확인할 수 있습니다. 마우스로 각 노드에 호버하면 해당 요소가 강조됩니다.

💡 팁: 마우스를 움직여 각 노드를 하이라이트하세요. 파란색=div, 초록색=p, 주황색=span

const root = document.documentElement; // <html> 루트
const body = document.body; // <body> 요소
const nodes = document.childNodes; // 자식 노드 목록

4.2 요소 선택과 탐색

DOM 조작의 첫 단계는 조작할 요소를 선택하는 것입니다. 다양한 선택 메서드와 탐색 방법이 있습니다.

선택 메서드

// 선택 메서드들
document.getElementById('id') // ID로 선택 (가장 빠름)
document.querySelector('selector') // CSS 선택자로 첫 요소 선택
document.querySelectorAll('selector') // CSS 선택자로 모든 요소 선택
document.getElementsByClassName('cls') // 클래스로 선택
document.getElementsByTagName('div') // 태그명으로 선택

노드 탐색

const elem = document.querySelector('.target');
elem.parentNode; // 부모 노드
elem.childNodes; // 자식 노드 목록 (텍스트 노드 포함)
elem.children; // 자식 요소 (요소만)
elem.firstChild; // 첫 자식
elem.nextSibling; // 다음 형제 노드
elem.previousSibling; // 이전 형제 노드

라이브 DOM 트리 뷰어

버튼을 클릭하여 DOM 구조를 동적으로 변경하고 실시간으로 트리를 확인하세요.

4.3 이벤트 시스템

이벤트는 사용자 상호작용(클릭, 입력 등)이나 브라우저 작동에 의해 발생하는 신호입니다. JavaScript는 이벤트를 감지하고 처리합니다.

이벤트 리스너 등록

element.addEventListener('click', (event) => {
  console.log('클릭됨!', event);
});

element.removeEventListener('click', handler);

이벤트 버블링과 캡처링

버블링은 이벤트가 발생한 요소에서 시작하여 부모 요소로 전파되는 것입니다. 캡처링은 루트에서 시작하여 하위 요소로 전파됩니다.

이벤트 버블링 데모

박스를 클릭하면 이벤트가 어떻게 전파되는지 확인하세요. 순서대로 불이 켜집니다.

이벤트 위임 (Event Delegation)

많은 자식 요소에 각각 리스너를 등록하는 대신, 부모 요소에 하나의 리스너를 등록하고 event.target을 확인하는 방식입니다.

parent.addEventListener('click', (event) => {
  if (event.target.matches('.child-item')) {
    // 자식 요소 클릭 처리
  }
});

4.4 DOM 조작 메서드

요소를 선택한 후, 내용을 변경하거나 새로운 요소를 추가할 수 있습니다.

메서드 설명 성능
innerHTML 요소의 HTML 내용을 문자열로 설정/조회 느림 (리파싱 발생)
textContent 요소의 텍스트만 설정/조회 (태그 미파싱) 빠름
createElement() 새로운 요소 생성 빠름
appendChild() 자식 요소 추가 중간
insertBefore() 특정 위치에 요소 삽입 중간
removeChild() 자식 요소 제거 빠름
// 요소 생성 및 추가
const newDiv = document.createElement('div');
newDiv.textContent = '새로운 요소';
newDiv.className = 'new-item';
parent.appendChild(newDiv);

// 요소 제거
parent.removeChild(newDiv);

// HTML 직접 변경 (주의: XSS 위험)
element.innerHTML = '<p>새 내용</p>';

4.5 가상 DOM 개념

실제 DOM 조작은 비용이 높으므로 (리플로우, 리페인트), React 같은 라이브러리는 가상 DOM을 사용합니다.

가상 DOM의 장점

  • 배칭(Batching): 여러 업데이트를 모아서 한 번에 적용
  • Diffing: 변경된 부분만 찾아 효율적으로 업데이트
  • 최적화: 불필요한 리렌더링 방지
  • 추상화: 개발자가 선언적 코드 작성 가능

실제 DOM vs 가상 DOM

1000개의 리스트 항목을 업데이트할 때의 차이를 시뮬레이션합니다.

Diffing 알고리즘

// 간단한 Diffing 개념
function diff(oldVDOM, newVDOM) {
  const changes = [];
  if (oldVDOM.type !== newVDOM.type) {
    changes.push({ action: 'replace', element: newVDOM });
  }
  // props, children 비교 ...
  return changes;
}

핵심 요약

  • ✅ DOM은 HTML 문서의 객체 표현이며 트리 구조로 되어있습니다.
  • ✅ querySelector, getElementById 등으로 요소를 선택하고 탐색합니다.
  • ✅ addEventListener로 이벤트를 감지하고 event 객체를 활용합니다.
  • ✅ createElement, appendChild 등으로 동적으로 요소를 조작합니다.
  • ✅ 가상 DOM은 성능 최적화를 위해 실제 DOM 업데이트를 스케줄링합니다.

한 줄로 이해하기

DOM은 브라우저가 HTML을 읽어 메모리에 만든 "조작 가능한 객체 트리"이고, JS는 이 트리를 바꿔 화면을 갱신합니다.

왜 배워야 하나?

실무 사례로 (1) "좋아요" 버튼을 누르면 하트 색이 즉시 바뀌고 카운트가 1 증가, (2) 모달 창을 열고 닫는 클릭 인터랙션, (3) 폼 입력값을 검증해 에러 메시지를 빨갛게 표시하는 등 거의 모든 동적 UI가 DOM 조작으로 만들어집니다. React·Vue도 결국 내부에서 DOM을 조작하므로 원리 이해는 필수입니다.

🌱 핵심 용어

  • 노드(Node): DOM 트리의 한 점 — 요소·텍스트·주석
  • 요소(Element): 태그로 만들어진 노드
  • 이벤트: 클릭·입력 같은 사용자/시스템 알림
  • 리스너: 이벤트가 일어났을 때 실행할 함수
  • 리플로우/리페인트: 레이아웃·픽셀을 다시 그리는 비싼 작업

개념 풀어쓰기

HTML이 도면이라면 DOM은 그 도면대로 지어 놓은 모형이고, JavaScript는 그 모형을 만지작거리는 손입니다. document.querySelector('.btn')은 "BTN이라는 이름표가 붙은 부품을 찾아줘"라는 명령이고, 찾은 부품에 .textContent = '저장됨'이라고 쓰면 글자만 바뀌고, .style.color = 'red'이라고 쓰면 색만 바뀝니다. 이벤트는 모형에 달린 작은 벨과 같아서, addEventListener('click', 함수)로 "이 벨이 울리면 이 함수를 실행해 줘"라고 약속해 둡니다. 부품을 새로 만들고 싶을 때는 createElement로 부품을 빚고 appendChild로 모형에 붙입니다. 단, 부품을 한꺼번에 100개씩 자주 바꾸면 모형 전체를 다시 그리느라 느려질 수 있으니, 한 번에 묶어서 바꾸는 습관이 중요합니다.

💡 학습 TIP: 크롬 개발자 도구(F12) → Elements 탭에서 직접 노드를 클릭하고 $0로 콘솔에서 조작해보면 즉시 반응을 볼 수 있어 이해가 빠릅니다.

한 페이지 요약

  • DOM = HTML의 객체 트리, JS로 동적 변경
  • 조회: querySelector·getElementById
  • 변경: textContent·classList·setAttribute
  • 이벤트: addEventListener('click', fn)
  • 변경은 묶어서 — 빈번한 조작은 성능 저하 원인
중 · 응용

이론 깊이

DOM 조작 비용은 (1) 자바스크립트 실행 → (2) 스타일 재계산 → (3) 레이아웃(리플로우) → (4) 페인트 → (5) 컴포지트 단계를 차례로 발생시킵니다. offsetHeight·getBoundingClientRect()처럼 레이아웃 정보를 읽는 동작은 강제 동기 레이아웃(Forced Synchronous Layout)을 유발할 수 있어, 읽기와 쓰기를 섞으면 같은 프레임에서 레이아웃이 여러 번 일어납니다. 이를 막으려면 모든 읽기를 먼저 묶고 그 다음 쓰기를 모아 처리하거나(Read/Write 분리), DocumentFragment·requestAnimationFrame·IntersectionObserver 등 비동기/배치 API를 활용해야 합니다. 이벤트는 캡처링→타깃→버블링 3단계로 흐르며, 위임(Event Delegation)은 부모 한 곳에서 자식들의 이벤트를 받는 패턴으로 동적으로 생성되는 항목에 특히 유용합니다.

핵심 코드 패턴

// 이벤트 위임 + DocumentFragment 배치 추가
const list = document.querySelector('#todo-list');

list.addEventListener('click', (e) => {
  const li = e.target.closest('li[data-id]');
  if (!li) return;
  if (e.target.matches('.delete')) li.remove();
});

const frag = document.createDocumentFragment();
items.forEach(item => {
  const li = document.createElement('li');
  li.dataset.id = item.id;
  li.textContent = item.title;
  frag.appendChild(li);
});
list.appendChild(frag); // 한 번의 리플로우

HTML5 Drag & Drop API(dragstart·dragover·drop)와 dataTransfer를 활용해 칸반 보드를 구현할 수 있고, 라이브러리 없이도 충분합니다.

실무 응용

실무에서는 (1) MutationObserver로 외부 스크립트가 삽입한 요소를 감지해 자동 처리, (2) ResizeObserver로 컨테이너 크기 변화에 반응, (3) scrollIntoView({ behavior: 'smooth' })로 부드러운 스크롤을 JS 한 줄로 구현합니다.

⚠️ 자주 하는 실수: innerHTML에 사용자 입력을 그대로 넣으면 XSS 공격 위험. 텍스트는 textContent를, HTML 삽입이 필요할 땐 sanitize 거치기. 또한 이벤트 리스너를 추가하고 제거하지 않으면 메모리 누수 발생.
상 · 심화

전문가 레벨

가상 DOM(Virtual DOM)은 React가 도입해 유명해진 패턴으로, 매 렌더 시 JSX 트리를 객체로 만들고 이전 트리와 diff한 뒤 최소 변경 사항만 실제 DOM에 반영합니다. 그러나 가상 DOM은 공짜가 아니어서, Diffing 비용·메모리 사용·재조정(Reconciliation)이라는 오버헤드가 있습니다. 그래서 Solid·Svelte·Vue 3의 컴파일러 기반 접근, 그리고 Signal 패러다임은 "필요한 부분만 직접 갱신"하는 방향으로 회귀했습니다. 브라우저 내부 렌더 엔진(Blink·WebKit·Gecko)은 DOM 트리와 별개로 Render Tree·Layer Tree를 유지하며, 합성(compositing) 단계에서 GPU 텍스처로 합쳐 화면에 표시합니다. contain: layout paint·content-visibility: auto 같은 격리 힌트를 주면 브라우저가 변경 영향 범위를 좁혀 성능을 끌어올릴 수 있습니다.

고급 패턴

// 미니 가상 DOM diffing
function diff(oldNode, newNode) {
  if (!oldNode) return { type: 'CREATE', newNode };
  if (!newNode) return { type: 'REMOVE' };
  if (oldNode.tag !== newNode.tag) return { type: 'REPLACE', newNode };
  const propPatches = diffProps(oldNode.props, newNode.props);
  const childrenPatches = [];
  const len = Math.max(oldNode.children.length, newNode.children.length);
  for (let i = 0; i < len; i++) {
    childrenPatches.push(diff(oldNode.children[i], newNode.children[i]));
  }
  return { type: 'UPDATE', propPatches, childrenPatches };
}

최신 트렌드

(1) Signals 기반 세밀 반응성(Solid, Vue 3.4 Vapor): 가상 DOM 없이 직접 DOM 노드에 의존성을 연결해 업데이트, (2) Web Components + Lit: 프레임워크 독립적으로 재사용 가능한 DOM 컴포넌트 표준화, (3) HTML Streaming + Islands Architecture(Astro, Qwik): SSR 결과를 빠르게 보내고 인터랙티브한 부분만 점진적 hydration.
