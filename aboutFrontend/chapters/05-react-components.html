5장. React와 컴포넌트

컴포넌트 기반 UI 라이브러리 React의 핵심 개념

5.1 React란?

React는 Facebook에서 개발한 선언적 UI 라이브러리입니다. 상태(state)에 따라 UI가 자동으로 업데이트되는 컴포넌트 기반 아키텍처를 제공합니다.

React의 특징

⚛️ Virtual DOM

메모리의 DOM을 먼저 업데이트하고 실제 DOM에 반영

🔄 One-way Data Flow

부모 → 자식으로의 단방향 데이터 흐름

⚙️ Component Based

UI를 재사용 가능한 컴포넌트로 분할

🎣 Hooks API

함수형 컴포넌트에서 상태와 생명주기 관리

// React의 핵심: 상태에 따른 UI 자동 렌더링
const UI = f(state); // UI는 상태의 함수

5.2 JSX 문법

JSX는 XML처럼 보이는 JavaScript 확장 문법입니다. 브라우저가 직접 읽을 수 없으므로 Babel을 통해 순수 JavaScript로 변환됩니다.

JSX → JavaScript 변환

JSX 코드

function Button() {
  return (
    <button className="btn">
      Click me
    </button>
  );
}

변환된 JavaScript

function Button() {
  return React.createElement(
    'button',
    { className: 'btn' },
    'Click me'
  );
}

JSX 규칙

  • ✅ className을 사용 (class는 예약어)
  • ✅ 중괄호 {}로 JavaScript 표현식 포함
  • ✅ 항상 하나의 root 요소로 감싸기
  • ✅ self-closing 태그 사용 가능 (<img />)
  • ✅ 주석은 중괄호 내에 {/* 주석 */}

5.3 함수형 컴포넌트

현대 React에서는 함수형 컴포넌트를 주로 사용합니다. Props를 입력받아 JSX를 반환하는 순수 함수입니다.

Props와 Children

// Props: 부모에서 자식으로 전달되는 불변 데이터
function Card({ title, children }) {

      <h2>{title}</h2>
      {children}
    </div>
  );
}

// 사용

5.4 Hook 기초

Hook은 함수형 컴포넌트에서 상태와 생명주기를 관리하는 특별한 함수입니다.

useState Hook

useState 카운터 데모

상태를 관리하고 변경하는 기본 Hook입니다.

0
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        증가
      </button>
    </>
  );
}

useEffect Hook

// 렌더링 후 부수 효과(side effect) 실행
useEffect(() => {
  console.log('컴포넌트 마운트됨');
  // 정리(cleanup) 함수
  return () => {
    console.log('컴포넌트 언마운트됨');
  };
}, []); // 의존성 배열

의존성 배열 (Dependency Array)

의존성 배열 실행 시점
[] 마운트 시에만 한 번
[count] count 변경될 때마다
없음 매 렌더링 후마다

3

useEffect 실행

4

상태 변경 감지

setState로 상태 업데이트

5

리렌더링

2-3 단계 반복

5.5 컴포넌트 생명주기

모든 React 컴포넌트는 생명주기를 거칩니다: Mount → Update → Unmount

생명주기 단계

📍

마운트 (Mount)

컴포넌트 생성 및 DOM에 추가. useEffect(fn, [])가 이 때 실행됨.

🔄

업데이트 (Update)

Props나 State 변경으로 리렌더링. useEffect(fn, [deps])로 특정 변경 감지.

🗑️

언마운트 (Unmount)

컴포넌트가 DOM에서 제거. useEffect의 cleanup 함수 실행.

5.6 컴포넌트 패턴

React에서 자주 사용되는 디자인 패턴들입니다.

Container/Presentational 패턴

Container (Smart)

// 로직 담당
function UserContainer() {
  const [user, setUser] =
    useState(null);
  useEffect(() => {
    fetchUser();
  }, []);
  return <UserDisplay
    user={user} />;
}

Presentational (Dumb)

// UI 담당
function UserDisplay({ user }) {
  if (!user) {
    return <div>로딩중...</div>;
  }
  return (
    <div>{user.name}</div>
  );
}

Custom Hook 패턴

// 로직을 재사용 가능한 Hook으로 추출
function useForm(initialValues) {
  const [values, setValues] = useState(initialValues);
  
  const handleChange = (e) => {
    const { name, value } = e.target;
    setValues(prev => ({ ...prev, [name]: value }));
  };
  
  return { values, handleChange };
}

// 사용
function LoginForm() {
  const { values, handleChange } = useForm({ email: '', password: '' });
}

한 줄로 이해하기

React는 화면을 "상태(state)의 함수"로 표현해, 상태가 바뀌면 자동으로 UI를 다시 그리는 라이브러리입니다.

왜 배워야 하나?

실무 사례로 (1) 페이스북·인스타그램·넷플릭스 같은 대규모 SPA의 UI 구성, (2) 한 컴포넌트(예: Button)를 사이트 전체에서 재사용해 유지보수성 향상, (3) 디자인 시스템 라이브러리(Material UI, Chakra) 기반 빠른 개발이 있습니다. 한국 IT 채용 공고의 절반 이상이 React 경험을 요구할 정도로 시장 점유율이 높습니다.

🌱 핵심 용어

  • 컴포넌트: UI 조각을 캡슐화한 함수
  • JSX: JS 안에서 HTML처럼 쓰는 문법
  • Props: 부모가 자식에게 내려주는 데이터
  • State: 컴포넌트가 자체적으로 관리하는 데이터
  • Hook: 함수형 컴포넌트에 상태/효과를 더하는 함수

개념 풀어쓰기

React 컴포넌트는 "재료(props)를 받아 요리(JSX)를 내놓는 부엌"이라 생각하면 쉽습니다. 같은 부엌이라도 들어오는 재료가 바뀌면 결과 요리도 자동으로 달라집니다. State는 부엌 안의 메모장 같은 것으로, 손님 수(useState(0))를 적어두면 손님이 한 명 더 올 때마다 메모를 고치는 동시에 메뉴판도 자동으로 갱신됩니다. JSX는 "요리 레시피를 HTML 모양으로 적어둔 메모"이고, Babel이라는 번역가가 이를 React.createElement라는 진짜 명령어로 바꿔 React에 전달합니다. 컴포넌트는 작은 부품(Atom) → 조합 부품(Molecule) → 페이지(Organism)로 키우면 재사용성이 높아져, 회사가 커져도 코드가 엉키지 않습니다.

한 페이지 요약

  • UI = f(state) — 상태가 바뀌면 React가 자동으로 리렌더
  • 함수형 컴포넌트 + Hook이 표준
  • Props는 읽기 전용, State는 setState로만 변경
  • JSX는 표현식이며 변수에 담기 가능
  • 리스트 렌더 시 key prop 필수
중 · 응용

이론 깊이

React 렌더링은 (1) 트리거(상태 변경) → (2) 렌더(컴포넌트 함수 호출) → (3) 커밋(DOM 반영) 3단계입니다. useState의 setter는 비동기적으로 큐에 쌓이고 React가 자동으로 배치(batching) 처리하기 때문에, 같은 핸들러 안에서 두 번 setState해도 리렌더는 한 번만 발생합니다. useEffect는 커밋 후 실행되며, 의존성 배열(deps)에 들어간 값이 바뀔 때만 재실행됩니다. cleanup 함수를 반환하면 다음 effect 실행 전·언마운트 시에 호출되므로, 이벤트 리스너·타이머 해제에 사용해야 메모리 누수가 없습니다. useMemo·useCallback은 참조 동일성을 유지해 자식 컴포넌트의 불필요한 리렌더를 막지만, 반대로 메모이제이션 자체에 비용이 들기 때문에 측정 후 적용해야 합니다.

핵심 코드 패턴

// Custom Hook으로 데이터 페칭 캡슐화
function useFetch(url) {
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);
  const [loading, setLoading] = useState(true);
  useEffect(() => {
    const ctrl = new AbortController();
    setLoading(true);
    fetch(url, { signal: ctrl.signal })
      .then(r => r.json()).then(setData)
      .catch(e => { if (e.name !== 'AbortError') setError(e); })
      .finally(() => setLoading(false));
    return () => ctrl.abort();
  }, [url]);
  return { data, error, loading };
}

상 · 심화

전문가 레벨

React 18 이후 도입된 Concurrent Renderer는 렌더 작업을 중단·재개·폐기할 수 있는 Fiber 아키텍처 위에서 동작합니다. useTransition·useDeferredValue로 우선순위가 낮은 업데이트를 양보(yield)할 수 있어, 무거운 리스트가 입력 응답성을 막지 않게 만듭니다. Suspense와 결합하면 데이터 로딩 중에도 부분적으로 UI를 보여주는 Streaming SSR이 가능합니다. 가장 큰 패러다임 전환은 React Server Components로, 서버에서 렌더된 컴포넌트는 클라이언트 번들에 포함되지 않아 JS 페이로드가 극적으로 줄어듭니다. 이는 Next.js App Router의 기반이며, "use client"·"use server" 디렉티브로 경계를 명시합니다. 내부적으로 Hook은 Fiber 노드의 연결 리스트로 저장되어 호출 순서가 같아야 정확히 매칭되므로, 조건문 안에서 Hook을 호출하면 안 됩니다.

// Server Component + Client Boundary (Next.js App Router)
// app/posts/page.tsx (Server)
import LikeButton from './LikeButton';
export default async function Posts() {

    <article key={p.id}>

// app/posts/LikeButton.tsx
'use client';
import { useOptimistic, useTransition } from 'react';
export default function LikeButton({ id, initial }) {
  const [likes, addOptimistic] = useOptimistic(initial);
  const [pending, start] = useTransition();
  return <button onClick={() => start(async () => {
    addOptimistic(likes + 1);
    await fetch(`/api/like/${id}`, { method: 'POST' });
  })}>❤ {likes}</button>;
}
