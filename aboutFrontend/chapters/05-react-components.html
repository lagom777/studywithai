5장. React와 컴포넌트

컴포넌트 기반 UI 라이브러리 React의 핵심 개념

5.1 React란?

React는 Facebook에서 개발한 선언적 UI 라이브러리입니다. 상태(state)에 따라 UI가 자동으로 업데이트되는 컴포넌트 기반 아키텍처를 제공합니다.

React의 특징

⚛️ Virtual DOM

메모리의 DOM을 먼저 업데이트하고 실제 DOM에 반영

🔄 One-way Data Flow

부모 → 자식으로의 단방향 데이터 흐름

⚙️ Component Based

UI를 재사용 가능한 컴포넌트로 분할

🎣 Hooks API

함수형 컴포넌트에서 상태와 생명주기 관리

// React의 핵심: 상태에 따른 UI 자동 렌더링
const UI = f(state); // UI는 상태의 함수

5.2 JSX 문법

JSX는 XML처럼 보이는 JavaScript 확장 문법입니다. 브라우저가 직접 읽을 수 없으므로 Babel을 통해 순수 JavaScript로 변환됩니다.

JSX → JavaScript 변환

JSX 코드

function Button() {
  return (
    <button className="btn">
      Click me
    </button>
  );
}

변환된 JavaScript

function Button() {
  return React.createElement(
    'button',
    { className: 'btn' },
    'Click me'
  );
}

JSX 규칙

  • ✅ className을 사용 (class는 예약어)
  • ✅ 중괄호 {}로 JavaScript 표현식 포함
  • ✅ 항상 하나의 root 요소로 감싸기
  • ✅ self-closing 태그 사용 가능 (<img />)
  • ✅ 주석은 중괄호 내에 {/* 주석 */}

5.3 함수형 컴포넌트

현대 React에서는 함수형 컴포넌트를 주로 사용합니다. Props를 입력받아 JSX를 반환하는 순수 함수입니다.

Props와 Children

// Props: 부모에서 자식으로 전달되는 불변 데이터
function Card({ title, children }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      {children}
    </div>
  );
}

// 사용
<Card title="안녕">
  <p>카드 내용</p>
</Card>

5.4 Hook 기초

Hook은 함수형 컴포넌트에서 상태와 생명주기를 관리하는 특별한 함수입니다.

useState Hook

useState 카운터 데모

상태를 관리하고 변경하는 기본 Hook입니다.

0
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        증가
      </button>
    </>
  );
}

useEffect Hook

// 렌더링 후 부수 효과(side effect) 실행
useEffect(() => {
  console.log('컴포넌트 마운트됨');
  // 정리(cleanup) 함수
  return () => {
    console.log('컴포넌트 언마운트됨');
  };
}, []); // 의존성 배열

의존성 배열 (Dependency Array)

의존성 배열 실행 시점
[] 마운트 시에만 한 번
[count] count 변경될 때마다
없음 매 렌더링 후마다

useEffect 라이프사이클 타임라인

1

초기 렌더링

컴포넌트 함수 실행 → JSX 반환

2

DOM 업데이트

브라우저가 화면 업데이트

3

useEffect 실행

마운트 또는 업데이트 후 실행

4

상태 변경 감지

setState로 상태 업데이트

5

리렌더링

2-3 단계 반복

5.5 컴포넌트 생명주기

모든 React 컴포넌트는 생명주기를 거칩니다: Mount → Update → Unmount

생명주기 단계

📍

마운트 (Mount)

컴포넌트 생성 및 DOM에 추가. useEffect(fn, [])가 이 때 실행됨.

🔄

업데이트 (Update)

Props나 State 변경으로 리렌더링. useEffect(fn, [deps])로 특정 변경 감지.

🗑️

언마운트 (Unmount)

컴포넌트가 DOM에서 제거. useEffect의 cleanup 함수 실행.

5.6 컴포넌트 패턴

React에서 자주 사용되는 디자인 패턴들입니다.

Container/Presentational 패턴

Container (Smart)

// 로직 담당
function UserContainer() {
  const [user, setUser] =
    useState(null);
  useEffect(() => {
    fetchUser();
  }, []);
  return <UserDisplay
    user={user} />;
}

Presentational (Dumb)

// UI 담당
function UserDisplay({ user }) {
  if (!user) {
    return <div>로딩중...</div>;
  }
  return (
    <div>{user.name}</div>
  );
}

Custom Hook 패턴

// 로직을 재사용 가능한 Hook으로 추출
function useForm(initialValues) {
  const [values, setValues] = useState(initialValues);
  
  const handleChange = (e) => {
    const { name, value } = e.target;
    setValues(prev => ({ ...prev, [name]: value }));
  };
  
  return { values, handleChange };
}

// 사용
function LoginForm() {
  const { values, handleChange } = useForm({ email: '', password: '' });
}

핵심 요약

  • ✅ React는 상태에 따라 UI를 자동 업데이트하는 선언적 라이브러리입니다.
  • ✅ JSX는 XML 문법의 JavaScript 확장으로 React.createElement로 변환됩니다.
  • ✅ 함수형 컴포넌트는 Props를 받아 UI를 반환하는 순수 함수입니다.
  • ✅ Hook(useState, useEffect)으로 상태와 생명주기를 관리합니다.
  • ✅ 컴포넌트는 Mount → Update → Unmount의 생명주기를 거칩니다.
  • ✅ Container/Presentational 패턴과 Custom Hook으로 로직을 재사용합니다.
