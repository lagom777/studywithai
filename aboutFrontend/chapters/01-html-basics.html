1장. HTML 기초 — 웹의 뼈대

1.1 HTML이란?

HTML(HyperText Markup Language)은 웹 문서의 구조와 내용을 정의하는 마크업 언어입니다. "마크업"이란 콘텐츠에 태그(tag)라는 레이블을 붙여 그 의미를 명시하는 것을 의미합니다.

HTML5는 현재 표준 버전으로, 의미 있는 구조(semantic structure)를 강조합니다. 문서의 각 부분이 무엇을 의미하는지 명확히 함으로써 브라우저, 검색 엔진, 스크린 리더 등이 콘텐츠를 올바르게 이해할 수 있습니다.

<!DOCTYPE html> <html lang="ko"> <head> <meta charset="UTF-8"> <title>페이지 제목</title> </head> <body> <h1>제목</h1> <p>단락</p> </body> </html>

1.2 시맨틱 태그 (Semantic Tags)

시맨틱 태그는 콘텐츠의 의미를 명확히 전달하는 HTML 요소들입니다. <div>처럼 의미 없는 범용 컨테이너 대신, 각 부분의 용도를 명시하는 태그를 사용합니다.

<header> — 사이트/섹션 헤더
<nav> — 네비게이션 메뉴
<main> + <article>
주요 콘텐츠
<aside>
보조 정보
<footer> — 사이트 푸터

주요 시맨틱 태그:

  • <header>: 소개 또는 탐색 정보
  • <nav>: 네비게이션 링크
  • <main>: 문서의 주요 콘텐츠
  • <section>: 주제별 콘텐츠 그룹
  • <article>: 독립적인 콘텐츠 (블로그 글, 뉴스 기사 등)
  • <aside>: 보조 정보 (사이드바 등)
  • <footer>: 푸터

1.3 폼과 입력 (Forms & Input)

폼은 사용자로부터 데이터를 수집하는 가장 중요한 메커니즘입니다. HTML5는 다양한 입력 타입을 지원합니다.

1.4 접근성 (Accessibility, a11y)

접근성은 장애가 있는 사용자를 포함한 모든 사용자가 웹 콘텐츠를 이용할 수 있도록 하는 것입니다. HTML은 접근성의 기초입니다.

접근성 모범 사례:
  • Alt 텍스트: 모든 이미지에 설명적인 alt 속성 추가
    <img src="logo.png" alt="회사 로고">
  • 의미 있는 링크: "여기를 클릭" 대신 명확한 링크 텍스트
    <a href="/guide">HTML 가이드</a>
  • 헤딩 구조: h1→h2→h3 순서대로 중첩
    스크린 리더가 문서 구조를 이해할 수 있도록
  • 라벨 요소: 폼 입력에 <label> 항상 사용
    <label for="email">이메일:</label><input id="email">
  • ARIA 속성: 복잡한 UI에 role, aria-label 등 추가

한 줄로 이해하기

HTML은 웹 페이지의 "뼈대"를 만드는 언어로, 콘텐츠가 어떤 의미를 갖는지 태그로 표시합니다.

왜 배워야 하나?

HTML 없이는 어떤 웹 페이지도 만들 수 없습니다. 실무 사례로는 (1) 회사 홈페이지의 메인 페이지 구성, (2) 블로그 글 작성 시 제목·본문·이미지 구조 정의, (3) 쇼핑몰의 상품 상세 페이지에서 가격·설명·리뷰 영역 구분이 있습니다. 검색엔진(SEO)과 스크린리더가 콘텐츠를 이해하는 출발점이 바로 HTML입니다.

🌱 핵심 용어

  • 태그(Tag): <p>처럼 꺽쇠로 감싼 표시
  • 요소(Element): 시작 태그 + 내용 + 종료 태그 전체
  • 속성(Attribute): href="..."처럼 태그에 부여하는 추가 정보
  • 시맨틱(Semantic): 의미를 명확히 전달하는 태그 사용 방식
  • DOCTYPE: 문서가 HTML5 표준임을 알리는 첫 줄 선언

개념 풀어쓰기

HTML 문서를 집을 짓는 일에 비유해봅시다. 먼저 <!DOCTYPE html>은 "이 건물은 한국 건축법(HTML5)을 따른다"고 선언하는 명패입니다. <html>은 집 전체를 감싸는 외벽이고, <head>는 보이지 않는 설계 정보(제목, 글꼴, 메타데이터)를 담는 다락방입니다. 반면 <body>는 사람이 실제로 머무는 거실과 방들의 집합입니다. 그 안에서 <h1>은 큰 간판, <p>는 일반 문장, <a>는 다른 방으로 이동하는 문, <img>는 벽에 걸린 그림입니다. 이렇게 각 태그는 콘텐츠의 "역할"을 알려주기 때문에, 검색엔진과 보조 기기들이 어디가 제목이고 어디가 본문인지 정확히 파악할 수 있습니다. 처음에는 외울 태그가 많아 보이지만, 실제로 자주 쓰는 것은 20개 남짓이며 나머지는 필요할 때 검색하면 됩니다.

💡 학습 TIP: 메모장이나 VS Code에 .html 파일을 만들고 더블클릭해 브라우저에서 열어보세요. 코드를 바꾸고 새로고침하는 사이클이 가장 빠른 학습법입니다.

한 페이지 요약

  • HTML은 웹 문서의 구조와 의미를 정의하는 마크업 언어
  • 태그는 시작/종료 쌍으로 구성되며 속성으로 부가 정보 전달
  • 시맨틱 태그(header, main, article)로 의미 명확화
  • 접근성을 위해 alt, label, 헤딩 순서 준수 필수
  • HTML5는 폼·미디어·시맨틱이 강화된 현재 표준
중 · 응용

이론 깊이

HTML 문서는 브라우저에 의해 파싱되어 DOM(Document Object Model) 트리로 변환됩니다. 파서는 토큰화(Tokenization)와 트리 구축(Tree Construction)이라는 두 단계를 거쳐 각 태그를 노드로 만들고 부모-자식 관계를 형성합니다. 이때 <script> 태그는 기본적으로 파싱을 차단하기 때문에, defer나 async 속성으로 로딩 전략을 제어해야 렌더링 성능이 떨어지지 않습니다. <meta charset="UTF-8">은 가능한 한 head 최상단에 둬야 인코딩 오해석을 막을 수 있고, <meta name="viewport">는 모바일 반응형의 출발점입니다. 폼은 required, pattern, minlength처럼 브라우저 단계에서 검증을 시도하지만, 보안상 서버 측 검증을 반드시 병행해야 합니다.

핵심 코드 패턴

<!-- 시맨틱 + 접근성을 동시에 챙긴 폼 -->
<form action="/api/signup" method="post" novalidate>
  <label for="email">이메일</label>
  <input id="email" name="email" type="email"
         autocomplete="email" required
         aria-describedby="email-help">
  <small id="email-help">회사 이메일을 입력하세요</small>

  <button type="submit">가입</button>
</form>

실무 응용

실제 프로젝트에서는 (1) Open Graph 메타 태그로 카카오톡·페이스북 미리보기 카드를 디자인하고, (2) JSON-LD 구조화 데이터를 삽입해 구글 검색 결과의 별점·가격을 노출시키며, (3) <dialog> 요소로 자바스크립트 의존도가 낮은 모달을 구현합니다. 대형 커머스에서는 상품 페이지마다 itemprop(microdata)을 추가해 가격 비교 사이트에 노출되도록 합니다.

⚠️ 자주 하는 실수: <div>로 모든 영역을 감싸는 "div 수프"는 SEO·접근성에 치명적입니다. 또한 <b>·<i> 대신 의미를 가진 <strong>·<em>을 사용하고, 이미지 alt를 빈 문자열로 두는 것은 장식용 이미지일 때만 허용됩니다.
상 · 심화

전문가 레벨

HTML 파서는 WHATWG 명세에 따라 상태 머신(State Machine) 기반으로 동작하며, 잘못된 태그도 가능한 한 복구(error recovery)하도록 설계되었습니다. 이 관용성 때문에 닫지 않은 태그도 렌더링되지만, 결과적으로 DOM 구조가 달라져 CSS 선택자나 JS 쿼리가 의도와 다르게 동작할 수 있습니다. 브라우저는 HTML을 받자마자 Preload Scanner를 가동해 이후 등장할 리소스를 선제 다운로드하므로, rel="preload"·rel="modulepreload"·rel="preconnect" 힌트를 head에 적절히 배치하는 것이 LCP(Largest Contentful Paint) 최적화의 핵심입니다. 또한 Shadow DOM과 결합한 Web Components는 스타일·구조 캡슐화를 제공해 디자인 시스템에 자주 활용되며, <template>·<slot> 요소가 그 기반입니다. 접근성 측면에서는 ARIA를 남용하지 말고 "First rule of ARIA: don't use ARIA"를 따라 네이티브 시맨틱을 우선해야 합니다.

고급 패턴

// Custom Element + Shadow DOM
class UserCard extends HTMLElement {
  static observedAttributes = ['name', 'role'];
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }
  connectedCallback() {
    this.render();
  }
  attributeChangedCallback() { this.render(); }
  render() {
    this.shadowRoot.innerHTML = `
      <style>:host{display:block;padding:1rem;border:1px solid #ccc}</style>
      <h3>${this.getAttribute('name')}</h3>
      <p>${this.getAttribute('role')}</p>
    `;
  }
}
customElements.define('user-card', UserCard);

최신 트렌드

(1) View Transitions API로 페이지 전환 애니메이션을 HTML/CSS만으로 구현 가능, (2) Declarative Shadow DOM은 SSR 환경에서도 Web Components 사용을 가능케 하며, (3) HTML Streaming(Transfer-Encoding: chunked)으로 React Server Components처럼 콘텐츠를 점진적으로 흘려보내 TTFB를 개선하는 패턴이 확산 중입니다.
