3장. JavaScript 핵심

3.1 변수와 타입

JavaScript는 동적 타입 언어로, 변수는 런타임에 어떤 타입이든 가질 수 있습니다. ES6부터 세 가지 변수 선언 방식이 있습니다.

  • var: 함수 스코프, 호이스팅됨, 재선언 가능 (레거시)
  • let: 블록 스코프, 호이스팅되지만 접근 불가, 재선언 불가 (권장)
  • const: 블록 스코프, 재할당 불가, 초기화 필수 (권장)
// 변수 선언 const name = "김철수"; // 문자열 let age = 28; // 숫자 var isActive = true; // 불린 // 타입 확인 console.log(typeof name); // "string" console.log(typeof age); // "number" console.log(typeof isActive); // "boolean" // 원시 타입 vs 참조 타입 const primitive = 42; // 원시: 값으로 복사 const reference = {id: 1}; // 참조: 주소로 복사

원시 타입(Primitive Types): string, number, boolean, null, undefined, symbol, bigint

참조 타입(Reference Types): object, array, function

3.2 함수와 클로저

함수는 JavaScript의 핵심입니다. 함수는 일급 객체로, 변수에 할당되거나 다른 함수의 인자로 전달될 수 있습니다. 클로저는 함수가 자신의 스코프 밖의 변수에 접근할 수 있는 현상입니다.

// 함수 선언 function add(a, b) { return a + b; } // 함수 표현식 const multiply = function(a, b) { return a * b; }; // 화살표 함수 (Arrow Function) const divide = (a, b) => a / b; const greet = (name) => `안녕하세요, ${name}!`; // 클로저 function makeCounter() { let count = 0; return function() { return ++count; }; } const counter = makeCounter(); console.log(counter()); // 1 console.log(counter()); // 2

클로저 시각화

클로저는 함수와 그 함수가 선언된 렉시컬 환경의 조합입니다. 내부 함수는 외부 함수의 변수에 접근할 수 있습니다.

Global Scope (전역 스코프)
const makeCounter = function() { ... }
const counter = makeCounter()
makeCounter Scope (함수 스코프)
let count = 0 ← 클로저가 기억하는 변수
Inner Function (반환된 함수)
return ++count

이 함수는 count 변수에 접근할 수 있습니다!

3.3 비동기 프로그래밍

JavaScript는 단일 스레드이지만 비동기 작업을 지원합니다. 콜백, Promise, async/await 세 가지 방식이 있습니다.

// 콜백 (Callback) - 콜백 지옥 문제 setTimeout(() => { console.log("1초 후"); }, 1000); // Promise - 상태 관리 const promise = new Promise((resolve, reject) => { setTimeout(() => resolve("완료!"), 1000); }); promise.then(result => console.log(result)); // async/await - 동기처럼 작성 async function fetchData() { try { const response = await fetch('https://api.example.com/data'); const data = await response.json(); return data; } catch (error) { console.error('에러:', error); } }

3.4 ES6+ 주요 기능

Destructuring (구조 분해)

// 배열 구조 분해 const [first, second, ...rest] = [1, 2, 3, 4, 5]; // 객체 구조 분해 const {name, age, city = "서울"} = {name: "김철수", age: 28}; // 함수 매개변수 구조 분해 const greet = ({name, age}) => { console.log(`${name}은 ${age}세입니다.`); };

Spread Operator (스프레드 연산자)

// 배열 펼침 const arr1 = [1, 2, 3]; const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5] // 객체 펼침 const obj1 = {a: 1, b: 2}; const obj2 = {...obj1, c: 3}; // {a: 1, b: 2, c: 3} // 함수 인자 펼침 const numbers = [1, 2, 3]; Math.max(...numbers); // 3

Template Literals (템플릿 리터럴)

const name = "김철수"; const age = 28; // 백틱으로 감싸고 ${} 사용 const message = `${name}은 ${age}세입니다.`; // 멀티라인 문자열 const html = ` <div> <h1>${name}</h1> <p>나이: ${age}</p> </div> `;

3.5 프로토타입과 클래스

JavaScript의 모든 객체는 프로토타입을 통해 다른 객체를 상속받습니다. ES6 클래스는 프로토타입 기반 상속을 더 쉽게 만들어줍니다.

// 프로토타입 기반 상속 function Animal(name) { this.name = name; } Animal.prototype.speak = function() { console.log(`${this.name}이 울었습니다.`); }; // ES6 클래스 class Dog extends Animal { constructor(name, breed) { super(name); this.breed = breed; } speak() { console.log(`${this.name}이 멍멍 울었습니다.`); } } const dog = new Dog("뽀삐", "골든 리트리버"); dog.speak(); // "뽀삐이 멍멍 울었습니다."

한 줄로 이해하기

JavaScript는 웹 페이지에 "동작"을 입히는 프로그래밍 언어로, 클릭·입력·통신 같은 사용자 인터랙션을 처리합니다.

왜 배워야 하나?

실무 사례로 (1) 쇼핑몰 장바구니의 수량 변경과 합계 자동 계산, (2) 로그인 폼의 실시간 유효성 검사, (3) 지도 위 마커를 드래그해 위치를 선택하는 기능 등 거의 모든 동적 동작이 JS의 영역입니다. 프론트엔드뿐 아니라 Node.js로 서버, React Native로 모바일 앱까지 만들 수 있어 한 언어로 많은 일을 해결할 수 있습니다.

🌱 핵심 용어

  • 변수: let·const로 값을 담는 상자
  • 함수: 재사용 가능한 코드 묶음
  • 객체/배열: 여러 값을 묶어 다루는 자료 구조
  • 이벤트: 클릭·키 입력 등 사용자 행동 알림
  • 비동기: 시간이 걸리는 작업을 기다리지 않고 처리

개념 풀어쓰기

JavaScript를 부엌 도우미 로봇으로 비유해보세요. let 재료 = "양파"는 "재료라는 통에 양파를 담는다"는 뜻이고, function 썰기() { ... }는 "썰기 동작 매뉴얼"을 정의하는 일입니다. 매뉴얼을 만들어 두면 필요할 때마다 썰기()로 호출해 재사용할 수 있습니다. 객체는 "양파라는 카드에 색=흰색, 매운맛=강함을 같이 적은 메모"이고, 배열은 "양파 3개를 한 바구니에 담은 것"입니다. 이벤트 처리는 "주인이 종을 울리면(click) 로봇이 미리 약속한 일을 한다"는 식입니다. 비동기는 특히 중요해서, 라면을 끓이는 동안 도마 정리를 하듯 "기다리지 않고 다음 일을 하다가, 라면이 다 되면 알림을 받아 처리"하는 방식이 async/await입니다. 처음에는 어렵지만 콘솔(F12 → Console)에 직접 코드를 쳐보면 즉시 결과를 보면서 익숙해질 수 있습니다.

💡 학습 TIP: 책 한 권을 끝내려 하지 말고, 작은 토이 프로젝트(투두리스트, 계산기, 가위바위보)를 만들면서 필요한 문법을 그때마다 검색하세요.

한 페이지 요약

  • const 우선, 재할당 필요할 때만 let
  • 함수는 일급 객체 — 변수에 담고 인자로 전달 가능
  • 배열·객체 메서드(map·filter·reduce) 익히기
  • 이벤트 → 리스너 함수 → 상태 변경 → UI 업데이트 흐름
  • 비동기는 Promise·async/await로 동기적으로 읽히게 작성
중 · 응용

이론 깊이

JavaScript는 단일 스레드 이벤트 루프 기반 언어로, 콜 스택(Call Stack), 태스크 큐(Task Queue), 마이크로태스크 큐(Microtask Queue)가 협력해 비동기 작업을 처리합니다. Promise.then은 마이크로태스크로, setTimeout은 매크로태스크로 스케줄링되며 마이크로태스크가 항상 우선합니다. 클로저(Closure)는 함수가 선언될 당시의 렉시컬 환경을 기억하는 메커니즘으로 모듈 패턴, 메모이제이션, 커링의 토대가 됩니다. this는 호출 방식(메서드/함수/생성자/화살표)에 따라 결정되며, 화살표 함수는 자체 this가 없어 상위 스코프를 그대로 사용합니다. ES2015 이후 도입된 디스트럭처링·스프레드·템플릿 리터럴·옵셔널 체이닝·널리시 코얼레싱은 가독성을 크게 개선합니다.

핵심 코드 패턴

// 디바운스: 입력 끝나고 N ms 후에만 실행
function debounce(fn, wait = 300) {
  let timer;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), wait);
  };
}

// fetch + 에러 핸들링 + AbortController
async function getJSON(url, signal) {
  const res = await fetch(url, { signal });
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}

상 · 심화

전문가 레벨

V8 엔진은 인터프리터(Ignition)와 최적화 컴파일러(TurboFan, Maglev)로 코드를 실행하며, 모노모픽한 객체 형태(Hidden Class)를 유지할 때 가장 빠른 인라인 캐시를 사용합니다. 동일 객체에 동일 순서로 프로퍼티를 추가하고, 객체 형태를 도중에 바꾸지 않는 것이 성능에 유리합니다. 가비지 컬렉션은 세대별 마크-앤-스위프 방식으로, 짧게 사는 객체는 New Space에서, 오래 사는 객체는 Old Space에서 처리됩니다. 메모리 누수의 가장 흔한 원인은 분리된 DOM 노드를 클로저가 잡고 있는 경우와, 전역 이벤트 리스너 미해제입니다. Proxy·Reflect는 Vue·MobX 같은 반응성 시스템의 기반이며, WeakMap·WeakRef는 메모리 친화적 캐시 구현에 필수입니다. 또한 ES Modules의 정적 분석 가능 구조 덕분에 트리 셰이킹과 코드 스플리팅이 가능합니다.

고급 패턴

// Proxy 기반 반응성 시스템 (Vue 3 핵심 아이디어 단순화)
const targetMap = new WeakMap();
let activeEffect = null;

function reactive(obj) {
  return new Proxy(obj, {
    get(target, key) {
      track(target, key);
      const v = Reflect.get(target, key);
      return typeof v === 'object' && v !== null ? reactive(v) : v;
    },
    set(target, key, value) {
      const ok = Reflect.set(target, key, value);
      trigger(target, key);
      return ok;
    }
  });
}
function effect(fn) { activeEffect = fn; fn(); activeEffect = null; }
function track(t, k) { /* deps 등록 */ }
function trigger(t, k) { /* deps 실행 */ }
