2장. CSS와 레이아웃

2.1 CSS 기초

CSS(Cascading Style Sheets)는 HTML 요소의 모양, 레이아웃, 애니메이션을 정의합니다. "캐스케이딩"이란 여러 스타일 규칙이 우선순위에 따라 적용되는 것을 의미합니다.

선택자(Selectors)

/* 요소 선택자 */ p { color: blue; } /* 클래스 선택자 */ .highlight { background: yellow; } /* ID 선택자 */ #header { font-size: 24px; } /* 조합 선택자 */ div > p { margin: 10px; } div p { padding: 5px; } /* 속성 선택자 */ input[type="email"] { border: 2px solid blue; } /* 의사 클래스 */ a:hover { color: red; } li:first-child { font-weight: bold; }

특이성(Specificity)

특이성은 어떤 CSS 규칙이 적용될지 결정합니다. 점수가 높을수록 높은 우선순위를 가집니다:

  • ID 선택자: 100점
  • 클래스, 속성, 의사 클래스: 10점
  • 요소 선택자: 1점

하 · 입문

한 줄로 이해하기

CSS는 HTML 요소를 어떻게 보여줄지 결정하는 "스타일 규칙"이며, 색·크기·배치를 모두 담당합니다.

왜 배워야 하나?

실무 사례로는 (1) 회사 브랜드 컬러를 사이트 전반에 일관되게 적용, (2) 모바일·태블릿·PC에서 자동으로 레이아웃이 바뀌는 반응형 디자인, (3) 호버 시 버튼이 살짝 떠오르는 마이크로 인터랙션 구현이 있습니다. 잘 짠 CSS는 코드 양을 절반으로 줄이고, 디자이너와의 협업을 부드럽게 만듭니다.

🌱 핵심 용어

  • 선택자(Selector): 어떤 요소에 스타일을 적용할지 지정
  • 속성/값(Property/Value): color: red;처럼 무엇을 어떻게 바꿀지
  • 박스 모델: content + padding + border + margin 4겹
  • Flexbox/Grid: 1차원·2차원 레이아웃 시스템
  • 캐스케이드: 여러 규칙이 겹쳤을 때 우선순위 결정 방식

개념 풀어쓰기

HTML이 집의 골조라면 CSS는 인테리어입니다. 똑같은 골조여도 페인트 색, 가구 배치, 조명 밝기에 따라 분위기가 완전히 달라지듯, CSS만 바꿔도 동일 HTML이 미니멀 디자인이 되기도 하고 화려한 카드 갤러리가 되기도 합니다. CSS 규칙은 "선택자 { 속성: 값; }" 형태로 단순합니다. 예를 들어 p { color: blue; }는 "모든 p 요소의 글자색을 파랑으로"라는 뜻입니다. 박스 모델은 모든 요소가 양파처럼 4겹의 영역을 가진다는 모델인데, 이를 이해하지 못하면 "왜 여백이 의도와 다르게 생기지?" 같은 문제로 며칠을 헤맬 수 있습니다. Flexbox는 한 줄로 정렬하기 좋고, Grid는 신문처럼 2차원으로 배치할 때 위력을 발휘합니다. 처음엔 두 가지를 동시에 배우려 하지 말고 Flex부터 손에 익히세요.

💡 학습 TIP: Flexbox Froggy, Grid Garden 같은 게임형 학습 사이트로 30분만 놀아도 핵심 속성이 손가락에 새겨집니다.

한 페이지 요약

  • CSS는 선택자·속성·값의 조합으로 스타일을 적용
  • 박스 모델은 content/padding/border/margin 4단계
  • Flexbox = 1차원, Grid = 2차원 레이아웃
  • 캐스케이드와 명시도(specificity)가 우선순위 결정
  • 미디어 쿼리로 반응형 구현, 모바일 우선 권장
중 · 응용

이론 깊이

CSS의 핵심은 "캐스케이드(Cascade) + 명시도(Specificity) + 상속(Inheritance)" 세 축입니다. 동일 속성에 여러 규칙이 겹치면 (1) 출처(작성자/사용자/브라우저), (2) !important, (3) 명시도 점수(인라인=1000, ID=100, 클래스=10, 태그=1), (4) 작성 순서 순으로 결정됩니다. box-sizing: border-box를 전역으로 설정하면 padding이 width를 침범하지 않아 레이아웃 계산이 직관적이 됩니다. Flexbox는 justify-content(주축)와 align-items(교차축)의 조합이 핵심이고, Grid는 grid-template-areas로 영역을 ASCII 아트처럼 정의하는 방식이 가독성이 가장 좋습니다. 애니메이션은 transform과 opacity 위주로 작성해 GPU 합성 단계에서 처리되도록 해야 60fps를 유지할 수 있습니다.

핵심 코드 패턴

/* 카드 그리드: 자동으로 열 개수가 늘고 줄어듦 */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 1rem;
}

/* 호버 애니메이션: GPU 가속 */
.card {
  transition: transform .2s ease, box-shadow .2s ease;
  will-change: transform;
}
.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 12px 24px -8px rgba(0,0,0,.15);
}

실무 응용

실무에서는 (1) 디자인 토큰(색·간격·글꼴)을 CSS 커스텀 프로퍼티로 정의해 디자인 시스템을 구축, (2) Tailwind/CSS Modules로 클래스 충돌을 차단, (3) clamp(min, ideal, max)로 글자 크기를 화면 크기에 따라 부드럽게 조절합니다.

⚠️ 자주 하는 실수: margin 충돌(collapsing margins), z-index 남용으로 쌓임 맥락(stacking context) 무너뜨리기, 절대 단위(px)만 사용해 접근성 떨어뜨리기. 가능한 한 rem·em을 사용하세요.
상 · 심화

전문가 레벨

브라우저 렌더링 파이프라인은 Style → Layout → Paint → Composite 4단계로 이루어집니다. 너비/위치 변경은 Layout부터 다시 계산되어 비용이 크지만, transform·opacity는 Composite 단계만 거치므로 60fps에 적합합니다. 또한 Layer Promotion을 유발하는 will-change·transform: translateZ(0)을 남용하면 메모리 폭증으로 오히려 성능이 악화될 수 있어 측정 후 적용해야 합니다. 컨테이너 쿼리(Container Queries)는 컴포넌트가 자기 부모 너비에 따라 스스로 모양을 바꿀 수 있게 하여 미디어 쿼리 패러다임을 보완하며, :has() 부모 선택자는 마침내 CSS만으로 "자식 상태에 따라 부모 스타일 변경"을 가능케 했습니다. 캐스케이드 레이어(@layer)는 명시도 전쟁을 끝내는 도구이며, 색 함수 color-mix()·oklch()는 색채 인지 균형 잡힌 팔레트를 만들 때 핵심입니다.

고급 패턴

/* 컨테이너 쿼리 + :has() + 캐스케이드 레이어 */
@layer reset, base, components, utilities;

@layer components {
  .card { container-type: inline-size; }

  @container (min-width: 480px) {
    .card { display: grid; grid-template-columns: 120px 1fr; }
  }

  /* 자식 input이 invalid이면 부모 스타일 변경 */
  .field:has(input:invalid) {
    border-color: hsl(0 80% 60%);
  }
}

최신 트렌드

(1) View Transitions API: SPA 페이지 전환을 CSS만으로 모핑, (2) CSS Nesting: SCSS 없이도 네이티브 중첩 작성 가능(2023년 모든 메이저 브라우저 지원), (3) Subgrid: 자식 그리드가 부모의 트랙 라인을 상속해 카드 정렬이 픽셀 단위로 맞아떨어집니다.
